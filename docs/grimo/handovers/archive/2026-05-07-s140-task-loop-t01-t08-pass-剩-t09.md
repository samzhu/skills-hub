### Lessons Learned

- **Modulith verify cycle 是 pre-existing main branch 問題**：`shared.api.ValidationErrorResponse` (S098b3-2 commit `7c94f0c`) 引用 `skill.validation.ValidationFinding` 形成 `shared <-> skill` 模組依賴循環。`processTestAot` 跑 `ApplicationModules.verify()` 必 fail。**所有 backend test 都受影響** — 不只 S140。Workaround: `-x processTestAot`。Tech debt should be filed: 拆 `ValidationFinding` 至 `shared.api` 或新 NamedInterface。
- **WebMvcSlice + @EnableCaching context fail 是廣泛問題**：`SkillshubApplication` 標 `@EnableCaching`，slice 不載 `CacheAutoConfiguration` → `CacheAspectSupport.afterSingletonsInstantiated` 找不到 CacheManager → 所有 WebMvc slice fail。Fix per-test：加 `@TestConfiguration` 提供 `ConcurrentMapCacheManager`。Should be moved to `WebMvcSliceTestBase` so other slice tests don't repeat the workaround.
- **POC 不只驗 SDK 還要驗 threshold filter**：S140 POC 驗 `Random(hashCode)` 產 deterministic vector + 排序穩定（H1/H2/H3 PASS）但**沒驗 SemanticSearchService.SIMILARITY_THRESHOLD = 0.3 會把所有 stub embeddings 過濾掉**。T07 implementation surface 此 gap → 改 ctor inject + e2e override `-1.0`。教訓：POC 應 trace **完整數據流** (embedder → vector store → threshold → frontend)，而不只 cosine 計算。
- **Spec text drift 在 implementation 階段大量被抓到**：AC-3 redirect URL 路徑、AC-2 quality 8-dim、AC-6 sparkline 都對不上實際 frontend code。教訓：design 階段 §4 file plan 應實際 grep frontend 確認 rendering 細節，不只看 prototype HTML。
- **Playwright `--list` 是輕量結構驗證**：每 spec 用 `npx playwright test --list` 確認 tags + test discovery 正確，不需 boot real browser。完整 happy-path run 集中在 T09，避免 6 次重複 boot。Test discovery 偶有 transient vitest matchers conflict (`Symbol($$jest-matchers-object)` redefine error) — 重 run 即解。
- **e2e LAB mode 比 storageState 簡單很多**：T05 BDD 原規劃 Playwright storageState 預先 OAuth login，實作改 `oauth.enabled=false` LAB mode（LabSecurityFilter 自動注入 lab-user）— 對所有 auth-gated endpoint 通用，不限 publish。Storage state 應留給 future spec 驗 lazy auth gate UX 才需要。
- **Reset allowlist 表名要對齊真 Flyway schema**：spec §4.1 寫 `skill_aggregate / notification / request_board_entry / flag_record` 等 hypothetical 名稱，實際 Flyway 是 `skills / notifications / requests / flags`。應 grep migration 確認。
- **planning-tasks Phase 3 loop 在 implementing-task 限「ONE task at a time」下，每 cycle context cost 高**：每次 round-trip dispatch 重 load skill 全文（~5K tokens 兩個 skill）+ task file content。8 task × ~10K tokens overhead = 80K tokens 純 loop machinery。Future: explore batch mode for similar-shape tasks (T03-T08 都是 Playwright spec writing)。
