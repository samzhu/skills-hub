## BDD
```
AC-1 + AC-9（部分）

Given 開發環境跑 V17 migration 後（既存 3 筆 skill row：
  - skills.author = OAuth sub raw（"111161306..."）
  - skills.owner_id = OAuth sub raw
  - skills.acl_entries JSONB 含 ["user:<sub>:OWNER", "public:*:read"]）

When 跑 ./gradlew flywayMigrate（或 bootRun 自動觸發 V18）

Then
  - users 表存在，每個 distinct sub 有 1 row
    （id 符合 `u_[0-9a-f]{6}`、oauth_provider='google'、handle='user-<6char>'、
      email='pending-backfill-u_<6hex>@unknown.local'、name=NULL、created_at=now、last_seen_at=now）
  - skills.author / skills.owner_id 全部 UPDATE 為對應 user_id
  - skills.acl_entries JSONB 內 "user:<sub>:perm" 全 rewrite 為 "user:<user_id>:perm"
  - 二次跑 migration no-op（ON CONFLICT DO NOTHING + WHERE 過濾）
```
