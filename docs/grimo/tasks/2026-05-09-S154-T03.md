```

Scenario D: CurrentUserProvider.current() 回 platform user_id
Given Alice JWT sub=111161...
When 任何 controller 呼叫 currentUserProvider.current()
Then 回 CurrentUser(userId="u_a3f9c1", sub="111161...", name="Alice Chen", email="alice@...",
                    handle="alice", roles=..., groups=..., companyId=...)
And `userId` 是 platform user_id（不是 OAuth sub）— breaking change for caller
```

### Key findings

1. **Mocking strategy for CurrentUserProviderTest**：原 test 是 pure unit test（無 Spring），新 ctor 多一個 `UserUpsertService` 依賴。解法：build stub `UserUpsertService` 透過 `Mockito.when(stub.upsertFromOidc(...))` 回傳 `User.createNew(sub, "google", sub, email, name, sub, avatar, now)` —— `id == sub`，讓既有 assertion `user.userId() == "alice"` 全部維持 valid，不需大量改 assertion。

2. **`new CurrentUser(...)` × 21 處的 bulk migration**：用 `CurrentUser.synthetic(userId, roles, groups, companyId)` static factory 替代，sed bulk-replace 7 個 test 檔（一個用 FQN form 另外 manual fix）。Production `new CurrentUser(...)` 只剩 3 處（canonical 8-arg ctor）：CurrentUser.synthetic 內部 + CurrentUserProvider.fromJwt + fromOAuth2User。

3. **`@Autowired` ctor disambiguation**：T02 加進 `UserUpsertService` 的 `@Autowired` pattern 在 T03 套到 CurrentUserProvider 不需要（後者只有單一 ctor）。但 ctor 簽章從 2-arg → 3-arg 是 breaking — 既有 test 直接 `new CurrentUserProvider(props, anomalyMetrics)` 全部 fail；CurrentUserProviderTest 是唯一受影響者，已 fix。
