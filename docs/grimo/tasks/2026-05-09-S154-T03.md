# S154-T03: Auth refactor — CurrentUserProvider 改回 user_id + MeController hook UPSERT

## Spec
S154 — Author Display Identity (Backend)

## BDD
```
AC-2（/me UPSERT）+ AC-9（placeholder email 覆寫）

Scenario A: 第一次登入觸發新 user UPSERT
Given Alice 第一次用 Google 登入（JWT sub=111161..., email=alice@..., name="Alice Chen"）
When 呼叫 GET /api/v1/me
Then users 表新增 1 row（id=u_<6hex>, oauth_provider='google', sub='111161...',
                       email='alice@...', name='Alice Chen', handle='alice'）
And response JSON 含 `userId="u_<6hex>"` + `handle="alice"`（既有 9 keys 之外）

Scenario B: 第二次登入 refresh 既有 user
Given Alice 既有 users row（last_seen_at=昨天）
When 呼叫 /me（OAuth name 改成「Alice Liu」）
Then users row 同 id，但 name 更新為 "Alice Liu"，last_seen_at refresh

Scenario C: V18 backfill placeholder email 被真登入覆寫
Given V18 backfill 後 Alice users row（email="pending-backfill-u_xxx@unknown.local"）
When Alice 第一次真登入觸發 /me
Then users row 同 id（oauth_provider+sub UNIQUE 中），email 更新為 "alice@..."，name 更新為 "Alice Chen"

Scenario D: CurrentUserProvider.current() 回 platform user_id
Given Alice JWT sub=111161...
When 任何 controller 呼叫 currentUserProvider.current()
Then 回 CurrentUser(userId="u_a3f9c1", sub="111161...", name="Alice Chen", email="alice@...",
                    handle="alice", roles=..., groups=..., companyId=...)
And `userId` 是 platform user_id（不是 OAuth sub）— breaking change for caller
```
