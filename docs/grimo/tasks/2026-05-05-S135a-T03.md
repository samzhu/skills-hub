## Implementation Notes

- 新 module `score/`（per Modulith pattern；不塞 skill/）
- `V15__create_skill_scores.sql` schema 完整 ddl 在 spec §4.5 + META §5.1
- `SkillScore.java`：implements `Persistable<String>` + `@Transient boolean isNew = false` + `@PersistenceCreator private SkillScore()` + factory `of(...)`（per `SkillVersion` pattern in V12）
- `QualityAxis.java`：`enum { VALIDATION, IMPLEMENTATION, ACTIVATION }`；DB column 為 VARCHAR(20)，靠 Spring Data 預設 enum-to-string 處理
- `SkillScoreRepository.java`：extends `ListCrudRepository<SkillScore, String>` + 3 個 derived/`@Query`：
  - `boolean existsBySourceEventId(String)` — derived
  - `List<SkillScore> findLatestBySkillId(String)` — `@Query("SELECT DISTINCT ON (axis) * FROM skill_scores WHERE skill_id = :skillId ORDER BY axis, evaluated_at DESC")`
  - `List<SkillScore> findLatestBySkillVersionId(String)` — 同模式
- `dimensions` JSONB 欄位用既有 `MapJsonbConverter`（`shared/persistence/JdbcConfiguration.java:61-69` 已註冊）— 不用再 wire
- Idempotency 走 ON CONFLICT DO NOTHING — 但注意 Spring Data JDBC `save()` 預設不發 ON CONFLICT（INSERT failure 才反應）。對 deterministic UUID PK + 用 `NamedParameterJdbcTemplate` 寫 raw INSERT 才能精確 ON CONFLICT 控制。可用 `@Modifying @Query` 或自寫 `saveIdempotent` method 在 repo（per AuditEventListener pattern）
  - **推薦做法**：repo 加 `@Modifying @Query("INSERT INTO skill_scores (...) VALUES (...) ON CONFLICT (id) DO NOTHING") int saveIdempotent(SkillScore)`，由 caller 顯式呼叫
- Test：`SkillScoreRepositoryTest extends RepositorySliceTestBase`（per S025b pattern；`@DataJdbcTest` slice 共用 cache key）
- AC tag `@DisplayName("AC-S135a-1: skill_scores DDL 與 idempotent INSERT")` 等（DDL/aggregate/repo 都歸 AC-1 第一階段；listener AC-1 後段在 T05）
