# S135a-T04: QualityJudge + RubricPrompts + 10-fixture Reproducibility Regression

## Spec
S135a — Backend Quality Score（LLM judge 核心；對 Implementation 4-dim + Activation 4-dim；per META §5.5；reproducibility 由 10+ fixture corpus 驗）

## BDD

```
Given SKILL.md body 內容（known-quality fixture 1 — high quality）
When QualityJudge.judgeImplementation(body)
Then return JudgeResponse with 4 DimensionScore
And dimension names ∈ {Conciseness, Actionability, WorkflowClarity, ProgressiveDisclosure}
And 每 dim score ∈ [0, 3]
And 每 dim reasoning 為 1-2 句敘事文字（非空）
And verdict 為 1 句 overall summary

Given SKILL.md description 內容（known-quality fixture 2）
When QualityJudge.judgeActivation(description)
Then return JudgeResponse with 4 DimensionScore
And dimension names ∈ {Specificity, Completeness, TriggerTermQuality, Distinctiveness}
And 每 dim score ∈ [0, 3]

Given fixture corpus（10+ known-score samples in src/test/resources/score-fixtures/）
When 對每 fixture 跑 QualityJudge × 5 次
Then per-dim score 5 次跑差異 ≤ 1 (integer scale)
And per-axis totalScore 5 次跑差異 ≤ 5 pt
And 與 fixture file 標註的 expected score 差異 ≤ 5 pt（calibration anchor）

Given Gemini API 回 invalid JSON（structured output 崩）
When QualityJudge.judgeImplementation()
Then throw RuntimeException 含 root cause（不 swallow — listener 才 retry）
```

## Implementation Notes

- `JudgeResponse.java` (record)：用 `@JsonClassDescription` + `@JsonPropertyDescription` annotation 讓 Spring AI BeanOutputConverter 自動產 JSON schema（per Spring AI LLM-as-Judge guide）
  - 含 nested record `DimensionScore(String dimension, int score, String reasoning)`
- `RubricPrompts.java`：static final String 兩個 — `IMPLEMENTATION_SYSTEM` 與 `ACTIVATION_SYSTEM`
  - 各含 4 個 dim 的 0-3 anchor 描述（per spec §3 BDD「0=missing, 1=weak, 2=adequate, 3=excellent」+ dim-specific judgment criteria）
  - 結尾必加 `Return ONLY valid JSON matching the schema. No markdown fences. No explanation outside JSON.`（per Spring AI guide）
  - 1-2 個 inline few-shot example（system prompt 內），對齊 sub-agent #1 建議「不與 Native Structured Output 衝突的方式」
- `QualityJudge.java`：依 spec §4.2 code skeleton（Updated 2026-05-05 用 GoogleGenAi*）
  - constructor 注入 `GoogleGenAiChatModel`（M5 spring-ai-google-genai 包提供）
  - `evaluatorVersion()` 方法 return `"gemini-2.5-flash@2026-05-05-prompt-v1"`（後續 prompt 改動 → bump）
- `StubQualityJudge.java` (test profile)：simple deterministic stub return fixed JudgeResponse（test stack 不打 real Gemini）
  - `@ConditionalOnProperty("skillshub.quality.judge.enabled" havingValue="false")` —對齊 ScanOrchestrator LlmJudge `@ConditionalOnProperty` pattern
  - real `QualityJudge` 加 `@ConditionalOnProperty("skillshub.quality.judge.enabled" matchIfMissing=true)`
- Fixture corpus：`backend/src/test/resources/score-fixtures/` 至少 10 個 `.md` 文件 + 1 個 `expected-scores.json`（標 known-score for regression）
  - 高品質範例（~95）/ 中品質（~70）/ 低品質（~40）各 3-4 個
  - 內容可參考 既有 `skill-author` skill 的 SKILL.md（高品質）+ 自製不完整的 skill 模板（低品質）
- `QualityJudgeFixtureTest.java`：
  - `@SpringBootTest` 真打 Gemini API（不能 stub — 驗 reproducibility 必須真 LLM）
  - `@EnabledIfEnvironmentVariable(named = "GEMINI_API_KEY", matches = ".+")` 防 CI 沒 key 時跑
  - 每 fixture 跑 5 次，計算 score variance；assertion 檢查上述 BDD 條件
  - **CI gate**：若 GEMINI_API_KEY env var 沒設，test 跳過但 build 不 fail（避免 CI block）
  - 本機 dev 跑：`GEMINI_API_KEY=... ./gradlew test --tests QualityJudgeFixtureTest`
- AC tag `@DisplayName("AC-S135a-8: LLM judge reproducibility")` / `@Tag("AC-S135a-8")`
