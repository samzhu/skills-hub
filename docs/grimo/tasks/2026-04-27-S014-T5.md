## 工作明細

### 1. application.yaml — 完整 datasource + flyway + vectorstore.pgvector 設定

詳見 spec §4.8。重點：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/mydatabase
    username: myuser
    password: secret
    hikari:
      maximum-pool-size: 10
  flyway:
    enabled: true
    baseline-on-migrate: false
    locations: classpath:db/migration
  ai:
    vectorstore:
      pgvector:
        initialize-schema: false       # Flyway V1 已建表
        schema-validation: false       # 我們的 vector_store 多了 owner / skill_id 欄位
        schema-name: public
        table-name: vector_store
        index-type: HNSW
        distance-type: COSINE_DISTANCE
        dimensions: 768
        max-document-batch-size: 10000
```

> **移除** `spring.mongodb.uri` 與 `spring.data.mongodb.auto-index-creation`（既有 yaml 應有）。

### 2. application-local.yaml — 清掉 firestore.enabled=false

```diff
 spring:
   docker:
     compose:
       enabled: true
       lifecycle-management: start-and-stop

   autoconfigure:
     exclude:
       - org.springframework.ai.model.google.genai.autoconfigure.embedding.GoogleGenAiEmbeddingConnectionAutoConfiguration
       - com.google.cloud.spring.autoconfigure.core.GcpContextAutoConfiguration

-        enabled: false
```

> Firestore 設定不再需要 — `FirestoreVectorStore` 仍會用 google-cloud-firestore SDK，但其 client 由 `SearchConfig` 條件性建立。

### 3. application-gcp.yaml — Cloud SQL Enterprise + PG18 + db-f1-micro + Private IP

詳見 spec §4.9 完整版。**規格鎖定**：Cloud SQL Enterprise + PostgreSQL 18 + db-f1-micro（shared core, 1 vCPU / 0.614 GB RAM）。

```yaml
spring:
  datasource:
    # Cloud SQL Enterprise + PG18 + db-f1-micro（max_connections=25）
    # Private IP（10.x.x.x）+ Cloud Run Direct VPC egress
    url: jdbc:postgresql://${SKILLSHUB_DB_HOST}:5432/${SKILLSHUB_DB_NAME:skillshub}
    username: ${SKILLSHUB_DB_USER}
    password: ${SKILLSHUB_DB_PASSWORD}
    hikari:
      # 25 (max) - 5 reserved = 20 ÷ 5 (Cloud Run max instances) = 4 → 取 3
      maximum-pool-size: 3
      minimum-idle: 1
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
      leak-detection-threshold: 60000   # f1-micro 連線少，洩漏要早點抓
```

> 不引入 `com.google.cloud.sql:postgres-socket-factory`；標準 JDBC URL 即可。
>
> Cloud SQL instance 建立 + Cloud Run 配置（屬 deployment 範圍，非本 task）：
> ```bash
> gcloud sql instances create skillshub-db \
>   --database-version=POSTGRES_18 \
>   --edition=enterprise \
>   --tier=db-f1-micro \
>   --region=$REGION \
>   --network=projects/$PROJECT_ID/global/networks/default \
>   --no-assign-ip \
>   --database-flags=cloudsql.enable_pgvector=on
>
> gcloud run services update skillshub \
>   --vpc-connector=projects/$PROJECT_ID/locations/$REGION/connectors/$CONNECTOR_NAME \
>   --vpc-egress=private-ranges-only \
>   --update-env-vars=SKILLSHUB_DB_HOST=10.x.x.x,SKILLSHUB_DB_NAME=skillshub
> # SKILLSHUB_DB_USER / SKILLSHUB_DB_PASSWORD 由 Secret Manager 注入（延續 S013 模式）
> ```
