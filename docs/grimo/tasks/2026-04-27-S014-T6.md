## 工作明細

### 1. 全套 grep 掃描，補修任何遺漏的 mongo 殘留

```bash
cd backend
grep -rn "@Document\|MongoRepository\|MongoTemplate\|@CompoundIndex\|spring.data.mongodb\|@DataMongoTest" src/
grep -rn "import org.springframework.data.mongodb" src/
```

預期上述 grep 應該全為 0 筆（除非：FirestoreVectorStore 自身內部使用 mongo client — 不該出現；若出現代表 T1-T5 漏抓）。

任何殘留 → 立即修；T6 不應該還有大改動，只是收尾。

### 2. ModularityTests 確認 shared/persistence module 邊界正確

```java
@Test
@DisplayName("AC-7: Spring Modulith 模組邊界（含 shared/persistence）仍綠")
@Tag("AC-7")
void verifyApplicationModuleBoundaries() {
    ApplicationModules modules = ApplicationModules.of(SkillshubApplication.class);
    modules.verify();
}
```

如 ApplicationModules.verify() 拋例外，依錯誤訊息修正：
- `shared.persistence` 應該是 `@ApplicationModule(displayName = "Shared Persistence")` 標註
- 任何 `skill` / `security` / `analytics` 對 `shared.persistence` 的依賴必須透過顯式 import（spring-modulith ModuleConfiguration）

### 3. 抽樣手動跑核心 API（生煙測試）

T6 完成 ./gradlew test 全綠後，再手動 bootRun：

```bash
cd backend && ./gradlew bootRun &
sleep 30   # 等 Flyway + ApplicationContext 啟動

# 1. 上傳測試 skill
curl -fsS -X POST http://localhost:8080/api/v1/skills/upload \
    -F "file=@test-skill.zip" \
    -F "version=1.0.0" \
    -F "author=t6-tester" \
    -F "category=DevOps"
# 應回 201 + JSON {"id": "..."}

# 2. 列表
curl -fsS http://localhost:8080/api/v1/skills | jq .
# 應回含剛上傳 skill 的 page

# 3. 詳情
curl -fsS http://localhost:8080/api/v1/skills/$ID | jq .
# 應回 SkillReadModel JSON

# 4. 加版本
curl -fsS -X PUT http://localhost:8080/api/v1/skills/$ID/versions \
    -F "file=@test-skill-v1.1.zip" \
    -F "version=1.1.0"
# 應回 200

# 5. 提交 flag
curl -fsS -X POST http://localhost:8080/api/v1/skills/$ID/flags \
    -H "Content-Type: application/json" \
    -d '{"type":"QUALITY","description":"test"}'
# 應回 201

# 6. 確認 vector_store 仍為空（FirestoreVectorStore 仍在主導）
psql -h localhost -p 5432 -U myuser -d mydatabase \
    -c "SELECT COUNT(*) FROM vector_store;"
# 應回 count=0

kill %1
```
