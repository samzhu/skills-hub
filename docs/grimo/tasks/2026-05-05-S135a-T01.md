## Implementation Notes

- 改 `backend/build.gradle.kts` — 找 `springAiVersion` property（gradle.properties 或 build.gradle.kts 本身）→ M4 → M5
- 不主動引入新 Spring AI artifact（spring-ai-google-genai 既有；BOM 升級 transitive 自動跟）
- M4 → M5 breaking changes per Phase 2 sub-agent #1（per META §5.5）：Options merging 移到 ChatClient.builder().combineWith()；ModelOptionsUtils.merge() 移除；SpringAiTestAutoConfigurations 移除 — 既有 SearchConfig / ScannerAiConfig 是否觸發須 compile 時驗
- 若 compile fail → 對齊 sub-agent #1 migration table：Options merging 改用 ChatClient builder；ModelOptionsUtils.merge() 改自寫 merge utility
- doc-sync 改兩檔：`docs/grimo/architecture.md` (AI: 行 + Framework Dependency Table 「Spring AI 2.0.0-M4」→「2.0.0-M5」) + 根 `CLAUDE.md` (Tech stack 段)
