# S142b-T02: SecurityReport endpoint (Service + Controller + 404 path)

## Spec
S142b — SkillDetailPage v2 Backend Supplement

## BDD

Given skill 含 `risk_findings` JSONB: 1 GITHUB_PAT (HIGH, secret) + 1 SENSITIVE_PATH_SSH (HIGH, pattern)
When  GET /api/v1/skills/{id}/security-report
Then  200 OK
And   response.checks.shell.status = "pass"
And   response.checks.paths.status = "fail" (1 HIGH)
And   response.checks.secrets.status = "fail" (1 HIGH)
And   response.checks.deps.status = "pass"
And   response.overall = "fail"
And   response.engineVersion = "risk-scanner v1.0" (constant)
And   response.ruleSetVersion = "2026-05" (constant)
And   response.scannedAt = SkillVersion.publishedAt (or risk_assessment.scanned_at if exists)
And   response.skillId / skillVersionId / skillVersion 對應 latest version

Given skill 已 publish 但 risk_findings 為 null（scan 未跑）
When  GET /api/v1/skills/{id}/security-report
Then  404 SECURITY_NOT_SCANNED
And   response.error = "SECURITY_NOT_SCANNED"
And   response.message = "Security report will be available shortly after publish."

Given skill 含 1 FORK_BOMB (HIGH, resource-dos) — outlier 折入
When  GET /api/v1/skills/{id}/security-report
Then  response.checks.shell.status = "fail"
And   shell.detail 含 "FORK_BOMB"

Given skill 含 1 PI_ROLE_JAILBREAK (HIGH, prompt-injection) + 1 META_OPACITY (MEDIUM, meta) + 1 LLM-generated finding
When  GET /api/v1/skills/{id}/security-report
Then  4 個 quad 都 status="pass"（無 findings 進入 4-quad）
And   skill.riskLevel 仍因 PI HIGH 升級為 HIGH（既有行為，不變）

Given skill 含 multiple findings 對應 paths quad
When  服務查詢 latest version 的 findings
Then  ?versionId={skillVersionId} query param 可指定特定版本（default = latest published）
