# S140-T01: Backend test-support infrastructure

## Spec
S140 — E2E Critical Path Backfill

## BDD
Given Spring Boot application running with `e2e` profile active
When  Playwright `_fixtures.ts` POSTs to `/internal/test/reset`, `/internal/test/seed/skill`, `/internal/test/seed/download-event`
Then  reset truncates 14 application data tables (Flyway history preserved)
And   seed/skill creates a Skill via `SkillCommandService.uploadSkill()` (full aggregate + outbox + audit path)
And   seed/download-event INSERTs N rows into `download_events` spread across `daysAgo` window
And   `EmbeddingModel` resolves to `e2eStubEmbeddingModel` (deterministic 768-dim Random) instead of NoOp / Google
And   under `default` / `prod` profile, `TestDataController` + `E2EEmbeddingConfig` beans do NOT exist (profile guard test verifies)
