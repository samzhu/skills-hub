### Key findings

1. **`@RequestParam("author")` drop 不需 caller 對齊**：Spring MVC 預設對 unknown request params silently ignore（沒 `@RequestParam` 接的就不解析）。既有 caller 仍可丟 `?author=alice` query string，server 不 reject 也不 read。實測：`SkillPublishForgeryTest.uploadWithForgedAuthorParam_serverOverridesToCurrentUser` 故意丟 `param("author", "u_alice_xx")`，controller 不爆 + service 收到 `BOB_USER_ID` 不是 `u_alice_xx`。

2. **JSON body forgery 比 query param 危險**：JSON body 反序列化會把 `"author"` 寫進 `CreateSkillCommand.author`，如果 controller 直接傳給 service 就被偽造。修法：controller rebuild CreateSkillCommand，**整體 author + authorNameSnapshot 都 server-override**，永不從 body 信任這兩個 field。

3. **`recordVersionPublished` null snapshot 語意 = "don't update"**：原想直接覆寫，但 OIDC name claim 偶爾會缺（spec §2.6）— 接 null 當「不變動」更安全，避免資料倒退。1-arg overload 內部就是 `(version, null)`。
