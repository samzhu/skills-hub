## Spec

## BDD

```

And result.errors() 含 "skill_md_line_count: SKILL.md has 600 lines (max 500)"

Given SKILL.md frontmatter name = "my--skill"（連續 hyphen）
When validate
Then result.errors() 含 "name: consecutive hyphens not allowed"

Given SKILL.md frontmatter name = "-leading-hyphen"
When validate
Then result.errors() 含 "name: must not start or end with hyphen"

Given SKILL.md frontmatter description = "  "（blank string）
When validate
Then result.errors() 含 "description: must not be blank"

Given SKILL.md frontmatter compatibility = ""（empty when provided）
When validate
Then result.errors() 含 "compatibility: must not be blank if provided"

Given SKILL.md frontmatter metadata.foo = 123（int 非 string）
When validate
Then result.errors() 含 "metadata: key 'foo' value must be a string"

Given SKILL.md frontmatter 後沒有 body content
When validate
Then result.errors() 含 "body_present: SKILL.md has no body content after frontmatter"

Given SKILL.md body 沒 "## Example" 也沒 code fence
When validate
Then result.valid() == true（warning 不擋 publish）
And result.warnings() 含 "body_examples: no example heading or code fence detected"

Given SKILL.md body 沒 numbered list 也沒 "## Steps" heading

Then result.warnings() 含 "body_steps: no step-by-step structure detected"

When validate
Then result.warnings() 含 "body_output_format: no output format guidance detected"

When BackEnd code 不修改
Then 編譯通過 + behavior 不變（既存 publish flow 不破）
And ValidationResult.of(valid, metadata, errors) 3-arg static factory 仍可用（warnings 預設 List.of()）
```

## Implementation Notes

- 改 `ValidationResult.java`：record 加第 4 個 component `List<String> warnings`；加 static factory `of(boolean, Map, List<String>)` 預設 warnings = List.of()，保留既有 3-arg 用法 zero-change
- 改 `SkillValidator.java`：
  - `NAME_REGEX` 由 `^[a-z0-9-]{1,64}$` 改 `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$` + 加 `name.contains("--")` check
  - `validateFieldConstraints` 加 description / compatibility blank guard
  - 新增 private methods: `validateLineCount` / `validateBodyPresent` / `validateMetadataFieldTypes` / `validateBodyExamples` (warning) / `validateBodySteps` (warning) / `validateBodyOutputFormat` (warning)
  - body 抽取：在 frontmatter 第二個 `---` delimiter 後的 content
  - validate() return 改回 `ValidationResult(valid, metadata, errors, warnings)` (4-arg)
- AC tagging：`@DisplayName("AC-S135a-5: ...")` / `@Tag("AC-S135a-5")`、`AC-S135a-6`、`AC-S135a-7`
- 不改 caller `SkillCommandService` — backward compat goal；若需要 expose warnings 到 publish response，留後續 spec
