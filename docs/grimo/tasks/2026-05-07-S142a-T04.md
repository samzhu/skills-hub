# S142a-T04: FileExplorerPanel (split-pane + scripts/ highlight + binary fallback)

## Spec
S142a — SkillDetailPage v2 Frontend Rework

## BDD

Given GET /api/v1/skills/{id}/files 回 file tree (含 SKILL.md / examples/ / tests/ / scripts/ / assets/)
When  FileExplorerPanel 渲染
Then  layout grid: 220px file tree + 1fr preview
And   height: calc(100vh - 300px)
And   min-height: 400px
And   container border + rounded 用 var(--line) / var(--r-lg)

Given file tree 含 folder children
When  user 點 folder
Then  toggle expand/collapse (預設全部展開 per prototype)
And   folder icon + folder name + size hint

Given `scripts/` directory in file tree
When  渲染
Then  目錄 row 加 ft-scripts-dir class — amber border-left + rgba(239,159,39,0.06) bg + amber-text color
And   名稱旁加「security scan」徽章 (ft-badge — amber-soft bg)

Given 檔案 path 開頭為 `scripts/`
When  渲染
Then  row 加 ft-in-scripts class — rgba(239,159,39,0.3) border-left + 淡 amber bg

Given user 點 file tree 內某個檔案
When  fetch GET /api/v1/skills/{id}/files/{*path}
Then  preview pane header 更新：path (mono) + lang badge (從副檔名 derive — md=Markdown / yml=YAML / sh=Bash / py=Python etc)
And   preview body 顯 file content (mono font, line-height 1.75)
And   syntax highlight 用 T01 shiki/prism wrapper (lang from badge)

Given user 點 scripts/install.sh
When  preview 渲染
Then  preview header 下方加 ft-security-banner (amber bg + 警示文字「This file is in scripts/ — scanned for security risks」)

Given binary file 點擊（PNG / image / 大檔）
When  preview 渲染
Then  body 顯 ft-binary fallback (icon + 「Binary file — preview unavailable」)
And   不嘗試 decode binary 為 text

Given file tree active item
When  user 點不同檔案
Then  active item border-left = accent + bg = rgba(127,119,221,0.1)
And   舊 active item state 清除

Given files endpoint 失敗 (network / 404)
When  FileExplorerPanel 處理
Then  顯 error fallback「檔案列表載入失敗」+ retry button
