## Key Design Notes

### SkillGrantService (core logic)
```java
@Transactional
public String grant(String skillId, GrantRequest req) {
    var actor = users.current().userId();
    var skill = skillRepo.findById(skillId).orElseThrow(() -> new NoSuchElementException(skillId));
    if (!actor.equals(skill.getOwnerId())) throw new NotSkillOwnerException();
    if (req.role() == Role.OWNER && grantRepo.existsBySkillIdAndRole(skillId, "OWNER")) {
        throw new OwnerAlreadyExistsException();
    }
    var grant = SkillGrant.create(skillId, req.principalType(), req.principalId(), req.role(), actor);
    grantRepo.save(grant);
    events.publishEvent(new SkillGrantedEvent(skillId, grant.id()));
    return grant.id();
}

@Transactional
public void revoke(String skillId, String grantId) {
    var actor = users.current().userId();
    var skill = skillRepo.findById(skillId).orElseThrow(() -> new NoSuchElementException(skillId));
    if (!actor.equals(skill.getOwnerId())) throw new NotSkillOwnerException();
    var grant = grantRepo.findById(grantId).orElseThrow(GrantNotFoundException::new);
    if ("OWNER".equals(grant.role()) && actor.equals(grant.principalId())) {
        throw new CannotRevokeOwnOwnerException();
    }
    grantRepo.deleteById(grantId);
    events.publishEvent(new SkillRevokedEvent(skillId, grantId));
}
```

### SkillGrantController 端點
```
POST   /api/v1/skills/{id}/grants       → 202 {"grantId": "..."}
DELETE /api/v1/skills/{id}/grants/{grantId} → 202
GET    /api/v1/skills/{id}/grants       → 200 [...]
```
- POST/DELETE 需 `@PreAuthorize("isAuthenticated()")` (owner 驗證在 service 層)
- GET 需 `@PreAuthorize("hasPermission(#id, 'Skill', 'read')")` (有讀取權才能看 grants)

### Exception → HTTP mapping (GlobalExceptionHandler 新增)
- `NotSkillOwnerException` → 403 `not_skill_owner`
- `OwnerAlreadyExistsException` → 409 `owner_already_exists`
- `GrantNotFoundException` → 404 `grant_not_found`
- `CannotRevokeOwnOwnerException` → 403 `cannot_revoke_own_owner`

注意：exceptions 放在 `skill.security` 包（不是 `shared.api`），避免 Modulith cycle。GlobalExceptionHandler 在 `shared.api` — 需在 `allowedDependencies` 加 `"skill :: security"` 或用 @ControllerAdvice 掃描。

實際上，Spring `@ControllerAdvice` 的 `@ExceptionHandler` 只需要 exception class 在 classpath，不需要顯式 Modulith dependency。GlobalExceptionHandler 可透過 class name string 或 catch by parent class。

**最簡設計**：exception class 放 `skill.security` package，GlobalExceptionHandler 用 `catch (RuntimeException ex)` + `ex.getClass().getSimpleName()` 識別，或用 message-based routing。

**替代設計**：將 exception class 放 `shared.api`（與 QualityNotEvaluatedException 同模式）。

選擇後者（shared.api）以對齊既有 QualityNotEvaluatedException pattern，避免 Modulith cycle。
