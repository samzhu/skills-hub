## 工作明細

### 1. SkillReadModel — @Document → @Table

詳見 spec §4.2 範例。所有欄位加 `@Column("snake_case")` 顯式對應；理論上 Spring Data JDBC 自動 camelCase ↔ snake_case，但顯式宣告較不脆弱。

### 2. SkillReadModelRepository

```java
public interface SkillReadModelRepository extends ListCrudRepository<SkillReadModel, String> {
    // T4 才加 @Modifying @Query incrementDownloadCount
}
```

> 既有 `MongoRepository` 沒有 derived query，遷移簡單。

### 3. SkillVersionReadModel — @Document + @CompoundIndex → @Table

```java
// Before
@Document("skill_versions")
@CompoundIndex(def = "{'skillId': 1, 'publishedAt': -1}", name = "idx_skillId_publishedAt")
public record SkillVersionReadModel(...) {}

// After
@Table("skill_versions")
public record SkillVersionReadModel(
    @Id String id,
    @Column("skill_id") String skillId,
    @Column("version") String version,
    @Column("storage_path") String storagePath,
    @Column("file_size") long fileSize,
    @Column("frontmatter") Map<String, Object> frontmatter,
    @Column("risk_assessment") Map<String, Object> riskAssessment,
    @Column("published_at") Instant publishedAt
) {}
```

### 4. SkillVersionReadModelRepository — 加 @Modifying @Query

```java
public interface SkillVersionReadModelRepository
        extends ListCrudRepository<SkillVersionReadModel, String> {

    List<SkillVersionReadModel> findBySkillIdOrderByPublishedAtDesc(String skillId);

    /**
     * S010 ScanOrchestrator 完成多引擎掃描後寫入 risk_assessment（T4 接上）。
     * 走 @Modifying @Query 而非 ScanOrchestrator 內 raw jdbc.update —
     * 原則：CRUD / single-row UPDATE 走 Spring Data JDBC、僅動態 query 才下 JdbcTemplate。
     */
    @Modifying
    @Query("""
        UPDATE skill_versions
           SET risk_assessment = CAST(:riskJson AS jsonb)
         WHERE skill_id = :skillId AND version = :version
        """)
    int updateRiskAssessment(
            @Param("skillId") String skillId,
            @Param("version") String version,
            @Param("riskJson") String riskJson);
}
```
