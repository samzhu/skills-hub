### Key findings

1. **AOT processor 對 multi-ctor 嚴格**：UserUpsertService 兩個 ctor（production single-arg + visible-for-test 3-arg）一開始導致 `processTestAot` FAIL — `No constructor or factory method candidate found`。修法：production ctor 加 `@Autowired` 顯式標記。對齊 dev-standards 「multiple ctor 用 @Autowired 消除歧義」隱性慣例（既有 SkillCommandService 等多 ctor service 也是同 pattern；本 task 後可加進 standards 文件）。

2. **`findByEmail` → `findFirstByEmail` 改名**：原本想用 `findByEmail` 對齊 spec naming，但 email 非 UNIQUE（同 email 跨 provider 可能；spec §2.6 預留），返回 `Optional<User>` 會在多 row 時拋 NonUniqueResultException。改 `findFirstByEmail` 顯式 single-result 語意（Spring Data `First` keyword）— UserResolver 對應改用 `findFirstByEmail`。

3. **`User.refreshFromOidc` null-safe semantics**：incoming null 不覆寫既有值（避免 OIDC 偶爾缺 claim 而資料倒退）。Email 無條件 sync（OIDC strict required）。Handle 不在此 path 更新（spec §2.6 mutability — handle 改走專屬 endpoint）。
