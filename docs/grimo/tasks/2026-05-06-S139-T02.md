### 設計差異 + 收穫

**fetchMe strict shape check 防 cascade fail**：
原本 fetchMe `(await res.json()) as AuthUser` 直接 cast，遇到 page test 的 wildcard
fetch mock 回 `[]`（空 array）時，user.sub === undefined → AuthArea charAt 炸 → 整個
AppShell render 空 → 18 個 page tests 連環 fail。Fix：fetchMe 加 strict check（typeof
data.sub !== 'string' → return null），mock-friendly 同時也是 production 防呆（backend
shape drift 不會讓 UI 整個炸）。**18 fail 因此一行 fix 全綠**。

**Backend MeController 待擴**（不在 T02 scope）：
spec §4.1 AuthUser 含 email/name/picture，但既有 MeController（per S027/S134）只回 `sub/
roles/groups/companyId/deptId/scope`。frontend 把 email/name/picture 設 optional 容錯；
T04 LAB OAuth deploy 後接 Google id_token，可在 backend 擴 MeController 從 JWT claims
帶出 email/name/picture（OIDC profile scope）。屬 nice-to-have，不阻 T03/T04 progression。

**Avatar dropdown 互動測 deferred E2E**：
Radix UI DropdownMenu 用 Portal + PointerEvent，jsdom + fireEvent.click 不友善（按 trigger
不會真的 mount Portal content）。本 unit test 驗 trigger structure（aria-haspopup="menu" /
img src）；dropdown 開 + 點 menu item 互動屬 Radix 自家責任，留 T04 LAB E2E smoke 覆蓋。
