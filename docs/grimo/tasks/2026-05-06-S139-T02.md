# S139-T02: Frontend auth core — `useAuth` hook + `AuthArea` + AppShell integration

## Spec
S139 — Login UI + Lazy Auth Gate

## BDD

```gherkin
Given frontend AppShell.tsx 目前 header 只有 logo + nav + bell（無 auth UI）
And   無 React hook 探測使用者登入狀態
And   bell 圖示無條件渲染（未登入用戶 fetchUnreadCount 會收 401，目前無人處理）

When  按 spec §4.1 / §4.2 / §5 落地：
        1. 新 api/auth.ts：fetchMe() 呼叫 GET /api/v1/me；200 → AuthUser；
           401 → null（其他 status throw）
        2. 新 hooks/useAuth.ts：React Query useQuery 包 fetchMe，30s
           staleTime；export login(returnTo?) → window.location =
           "/oauth2/authorization/skillshub?returnTo=" + encodeURIComponent(
           returnTo ?? location.pathname + location.search)；
           logout() → POST /logout (credentials: 'include') 後 reload "/"
        3. 新 components/AuthArea.tsx：
             - status === 'loading' → skeleton（h-8 w-20 muted）
             - status === 'anonymous' → <Button onClick={() => login()}>登入</Button>
             - status === 'authenticated' → shadcn DropdownMenu trigger 為
               圓形 avatar（user.picture URL 或 fallback 字母）；
               MenuItems：email（disabled）/ 我的技能 link / 登出 button
        4. AppShell.tsx：
             - 嵌入 <AuthArea /> 在 bell 右側
             - bell 改條件渲染：{ status === 'authenticated' && <Bell badge /> }
             - 移除 fetchUnreadCount 對 anonymous 的 401 噪訊

Then  AC-1 未登入訪問 /browse → AppShell 顯示「登入」按鈕；bell 不渲染
  And AC-4 已登入用戶頁面 → AppShell 顯示 avatar；點開 dropdown 看到
      email / 我的技能 / 登出
  And AC-5 dropdown「登出」按鈕 → POST /logout → reload → 回未登入狀態
  And AC-7 bell 圖示存在性 follows useAuth.status === 'authenticated'
```
