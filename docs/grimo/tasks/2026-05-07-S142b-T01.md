# S142b-T01: SecurityCategoryMapper pure unit

## Spec
S142b — SkillDetailPage v2 Backend Supplement

## BDD

Given a list of `SecurityFinding` rows from `risk_findings` JSONB
When `SecurityCategoryMapper.partition(findings)` 處理
Then 回傳 `Map<Category, List<SecurityFinding>>` 對應 4 quad (SHELL / PATHS / SECRETS / DEPS)
And `analyzer == "pattern"` AND `ruleId.startsWith("DANGEROUS_COMMAND_") OR "PIPE_TO_SHELL_"` → SHELL
And `analyzer == "resource-dos"`（FORK_BOMB 等）→ SHELL（折入）
And `analyzer == "pattern"` AND `ruleId.startsWith("SENSITIVE_PATH_")` → PATHS
And `analyzer == "secret"` → SECRETS
And `analyzer == "dep-vuln"` → DEPS
And `analyzer == "prompt-injection" / "meta" / "llm-judge"` → 不入 4 quad（return Map 不含這些 finding）

Given findings list per category
When `SecurityCategoryMapper.computeStatus(findings)` 處理
Then 任何 HIGH severity → "fail"
And 任何 MEDIUM severity（無 HIGH）→ "warn"
And empty 或全 LOW → "pass"

Given Map<Category, CheckStatus> categoryStatuses
When `SecurityCategoryMapper.computeSecurityScore(...)` 處理
Then 全 pass → 100
And 1 warn（無 fail）→ 75
And 2 warn（無 fail）→ 50
And 3+ warn（無 fail）→ 25
And 任何 fail → 25
And 全部 null（未掃描）→ return null

Given findings list 1 row 含 ruleId="GITHUB_PAT" message="Hardcoded GitHub PAT" filePath="scripts/install.sh" lineNumber=14
When `SecurityCategoryMapper.formatDetail(findings)` 處理
Then return "GITHUB_PAT · line 14: Hardcoded GitHub PAT"

Given findings list 3 rows 含 ruleIds [GITHUB_PAT, AWS_SECRET_KEY, OPENAI_KEY]
When formatDetail
Then return "3 findings: GITHUB_PAT, AWS_SECRET_KEY, OPENAI_KEY"

Given empty findings list
When formatDetail
Then return null
