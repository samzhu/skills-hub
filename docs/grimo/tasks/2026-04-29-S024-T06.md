**新增**：`backend/src/test/java/io/github/samzhu/skillshub/skill/query/SkillQueryControllerApiContractTest.java`

策略：
- `@SpringBootTest @AutoConfigureMockMvc @Import(TestcontainersConfiguration.class)`

- 對 GET /api/v1/skills/{id} / search / versions 各做 1 個 snapshot test：
  - assert response status 200
  - assert JSON 含特定 field set（用 JsonPath assertions for stable assertion）
  - assert NO `version` field expose（驗 `@JsonIgnore` 設定）

```java
@Test
@DisplayName("AC-11: GET /api/v1/skills/{id} JSON shape 與 v1.5.0 一致")
void apiContractRegression_findById() throws Exception {
    // setup: create skill via API
    var id = createTestSkill("test-skill", "alice", "DevOps");

    mockMvc.perform(get("/api/v1/skills/{id}", id))
        .andExpect(status().isOk())
        .andExpect(jsonPath("$.id").exists())
        .andExpect(jsonPath("$.name").value("test-skill"))
        .andExpect(jsonPath("$.description").exists())
        .andExpect(jsonPath("$.author").value("alice"))
        .andExpect(jsonPath("$.category").value("DevOps"))
        .andExpect(jsonPath("$.status").value("DRAFT"))
        .andExpect(jsonPath("$.latestVersion").doesNotExist())   // null 序列化
        .andExpect(jsonPath("$.riskLevel").doesNotExist())
        .andExpect(jsonPath("$.downloadCount").value(0))
        .andExpect(jsonPath("$.createdAt").exists())
        .andExpect(jsonPath("$.updatedAt").exists())
        .andExpect(jsonPath("$.aclEntries").isArray())
        .andExpect(jsonPath("$.version").doesNotExist());        // @JsonIgnore 過濾 internal lock
}
```

**新內容大綱**：
```markdown
## Architecture Pattern: Spring Data JDBC Rich Aggregate + Modulith Outbox (Core Domain)

> 本段於 v2.0.0（S024 ship 2026-04-29）改寫；前身為 ES + CQRS 模式（v1.4.0 為止）。
> 詳 ADR-002（Accepted 2026-04-29）。

### Core Concepts

**Skill aggregate**（`@Table("skills") extends AbstractAggregateRoot<Skill>`）
- @Id / @Version / @Column 等註解直接綁定 PostgreSQL skills 表
- 業務 method 充血：mutate state + registerEvent(domain event)
- repository.save(skill) 自動觸發 @DomainEvents → Modulith outbox INSERT 同 TX

**SkillVersion aggregate**（獨立 @Table("skill_versions")）
- 與 Skill 透過 plain String skillId FK 引用（不用 @MappedCollection 避開 delete-and-reinsert）
- attachRiskAssessment 充血方法 + ScanOrchestrator AFTER_COMMIT async 路徑

### Persistence

- skills 表 = Skill aggregate state（含 acl_entries jsonb；S016 設計保留）
- skill_versions 表 = SkillVersion aggregate state（含 frontmatter + risk_assessment + allowed_tools jsonb）
- domain_events 表 = audit log（由 AuditEventListener 寫入，async，at-least-once）
- event_publication 表 = Modulith outbox（投遞狀態追蹤）

### Domain Event Flow

```
Command → Service.@Transactional method:
  1. skillRepo.findById(id).orElseThrow()       # O(1) row read（vs ES O(events) replay）
  2. skill.businessMethod(cmd)                   # mutate state + registerEvent
  3. skillRepo.save(skill)                       # @DomainEvents proxy interceptor:
                                                 #   ① UPDATE skills SET ... WHERE id=? AND version=?
                                                 #   ② publish events → outbox INSERT 同 TX
  TX commit:
    - skills row UPDATED
    - event_publication row INSERTED（status NULL → 等待 listener）
  AFTER_COMMIT async:
    - SearchProjection 訂閱 → vector_store UPDATE
    - AnalyticsProjection 訂閱 → download_events INSERT (idempotent)
    - AuditEventListener 訂閱 → domain_events INSERT (audit log)
    - ScanOrchestrator 訂閱（SkillVersionPublishedEvent only）→ multi-engine scan pipeline
    - 各 listener 完成 → event_publication.status = 'COMPLETED'
```

### 既有 ES 模式遺跡

- v1.5.0 之前的 domain_events row 仍在 DB；audit log 路徑統一後新事件由 AuditEventListener 寫入；歷史 row 不變
- Skill.fromHistory(events) deprecated private static method 保留（emergency rollback；@Deprecated since v2.0.0）

### 跨模組依賴

[既有圖 — 無變動]
```

### 6.4 CLAUDE.md 改寫

**改 1 行**（L132 `## Architecture pattern` 段）：

從：
```
- **Core domain (skill, security):** Event Sourcing + CQRS
  - Command side: validate → produce domain event → persist to `domain_events` → publish
  - Query side: `@ApplicationModuleListener` consumes events → update read model projections
```

改為：
```
- **Core domain (skill):** Spring Data JDBC rich aggregate + Modulith outbox（per ADR-002 / S024 ship v2.0.0）
  - Aggregate method 充血：mutate state + registerEvent；service 端 3 行 orchestration（load → mutate → save）
  - `repo.save()` 透過 `@DomainEvents` 自動 publish 至 Modulith `event_publication` outbox（同 TX；at-least-once）
  - AFTER_COMMIT async listeners 訂閱 domain events 維護衍生資料（vector_store / download_events / domain_events audit log）
- **Security domain (security):** 仍採 ES + CQRS（暫無轉向計畫；流量低、events 簡單）
- **Supporting (search, analytics):** Read-side projections consuming domain events（不變）
- **Infrastructure (storage):** Traditional service (GCS operations)（不變）
- **Event bus:** Spring Modulith `ApplicationEventPublisher` + Event Publication Registry outbox
- **Audit log:** `domain_events` 表 PostgreSQL（由 `AuditEventListener` async 寫入；非 source of truth）
```

### 6.5 development-standards.md 改寫

**§「Event Sourcing + CQRS 規範」段（L21-32）整段改寫**為：

```markdown
### Spring Data JDBC Rich Aggregate + Modulith Outbox 規範（S024 起；per ADR-002）

- **Skill domain** 改用 Spring Data JDBC 充血聚合：`@Table` + `extends AbstractAggregateRoot` + `@Version` 樂觀鎖
- Aggregate method 充血：先驗證不變量（throw IllegalStateException 若違規）→ mutate state field → `registerEvent(...)`
- Service 層每命令方法 3-6 行 orchestration：`load → mutate → save`；無 `eventStore.save` / `events.publishEvent`
- `repository.save(aggregate)` 透過 Spring Data proxy interceptor `@DomainEvents` 自動 publish 至 Modulith `event_publication` outbox（同 TX）
- 跨 aggregate 一致性由 application service `@Transactional` 邊界內 `existsBy*` 預檢 + DB UNIQUE / FK constraint 兜底
- Aggregate 內**不**用 `@MappedCollection` / `AggregateReference` 引用其他 aggregate（per ADR-002 §2.3：delete-and-reinsert 雷）；用 plain `String foreignKey` 欄位
- 高頻寫子集合（如 ACL）用 `jsonb` 欄位行內 UPDATE，不拆獨立 aggregate
- `domain_events` 表為 audit log（非 source of truth）；新事件由 `AuditEventListener` async 訂閱寫入
- Domain event class 視為公開 API，**不得**任意改名（per deepwiki §3 陷阱 8 — 反序列化失敗）
- Domain event payload 只序列化 ID + 關鍵欄位，**不**含大型 SARIF / frontmatter（per deepwiki §3 陷阱 10 — 8191 byte limit）
- emergency rollback path：aggregate 保留 deprecated private static `fromHistory(events)` factory 從 audit log 重建 state；非正常使用路徑
```

保留 §「Spring Modulith Outbox 規範」段（S023 既有 — 10 條 validated patterns）不變。

## Anti-Patterns / Pitfalls

- **不要**在 architecture.md 完全刪掉 ES 段歷史 — 留個 short note 標記「v1.4.0 為止採 ES + CQRS；v2.0.0 起改 Spring Data JDBC + Modulith」+ 引用 ADR-002，給後人 audit trail
- **不要**改 PRD.md D20-D23 — ADR-002 supersede 機制設計就是「PRD 不動，ADR 紀錄變更」；改 PRD 違反 ADR 設計理念
- **不要**為 API contract snapshot 引入第三方 library（如 javers / approvaltests）— 用 jsonPath assertions 簡單可控；snapshot 鎖在 test 內字串
- 注意 `@JsonIgnore` 加在 `Skill.version` 欄位（@Version Long version）— Jackson 預設 serialize getter；若漏標前端會看到 internal lock 欄位
- ApplicationModulesTests 通常已存在於 baseline；T6 加 `@Tag("AC-13")` 即可，無需新建（如已存在）
- spec-roadmap.md 的 ES backlog 表可能位於 Backlog 段；找到再標 obsolete；M18 milestone 段也要更新
