# S024-T02: Skill 完整充血方法 + state machine 守護 + ACL JSONB inline

## Spec
S024 — Skill State-Based Aggregate Migration

## Goal
T1 已建 Skill aggregate skeleton（create + recordVersionPublished）。T2 補完 Skill 全部充血方法（suspend / reactivate / grantAcl / revokeAcl / recordDownload）+ state machine 守護（SUSPENDED 不能 publishVersion）+ ACL JSONB 行內 UPDATE（不 DELETE-and-reinsert 子表）。

## BDD

### AC-2 (full): Skill 全部充血方法 mutate state + register events
```gherkin
Given Skill aggregate（T1 已支援 create / recordVersionPublished）
When  skill.suspend(SuspendCommand) on PUBLISHED skill
Then  status = SUSPENDED、updatedAt 更新
And   skill.domainEvents() 含 SkillSuspendedEvent(aggregateId, reason, suspendedBy)

When  skill.reactivate(ReactivateCommand) on SUSPENDED skill
Then  status = PUBLISHED、updatedAt 更新
And   skill.domainEvents() 含 SkillReactivatedEvent(aggregateId, reason)

When  skill.recordDownload("alice")
Then  downloadCount += 1、updatedAt 更新
And   skill.domainEvents() 含 SkillDownloadedEvent (aggregateId, latestVersion, eventId)
```

### AC-6: state machine 守護 — SUSPENDED 不可 publishVersion
```gherkin
Given Skill skill1 已 publish v1.0.0 後 suspend (status=SUSPENDED)
When  呼叫 skill.recordVersionPublished("1.1.0")
Then  拋 IllegalStateException("Cannot publish version while skill is in SUSPENDED status")
And   skill 物件 state 不變（latestVersion 仍 "1.0.0"、status 仍 SUSPENDED）
And   skill.domainEvents() 不含新事件（state machine guard 在 mutation 前 throw）
```

### AC-8: grantAcl / revokeAcl 行內 UPDATE jsonb；無重複 grant 守護
```gherkin
Given Skill skill1 aclEntries = []
When  skill.grantAcl(GrantAclCommand("user", "alice", "read", "admin"))
Then  skill.aclEntries() = ["user:alice:read"]（getAclEntries 回不可變 view）
And   skill.domainEvents() 含 SkillAclGrantedEvent

Given skill1 aclEntries = ["user:alice:read"]
When  skill.grantAcl(GrantAclCommand("user", "alice", "read", "admin")) — 重複 grant
Then  拋 IllegalStateException("ACL entry already exists: user:alice:read")
And   skill.aclEntries() 不變、skill.domainEvents() 不含新事件

Given skill1 aclEntries = ["user:alice:read"]
When  skill.revokeAcl(RevokeAclCommand("user", "alice", "read", "admin"))
Then  skill.aclEntries() = []
And   skill.domainEvents() 含 SkillAclRevokedEvent

Given skill1 aclEntries = []
When  skill.revokeAcl(RevokeAclCommand("user", "alice", "read", "admin")) — 不存在 entry
Then  拋 IllegalStateException("ACL entry not found: user:alice:read")
```

### AC-7 (partial): VersionExistsException 由 service 處理（T4 完成）；T2 僅驗 Skill side 不重複保護
```gherkin
Given Skill 充血聚合**不**自管 publishedVersions set（per ADR-002 §2.3 — 由 SkillVersion aggregate + DB UNIQUE 兜底）
When  Skill.recordVersionPublished("1.0.0") 後再呼叫 Skill.recordVersionPublished("1.0.0")
Then  Skill 不拋（aggregate 內無 version uniqueness 知識）
And   重複 version 由 SkillVersionRepository.existsBySkillIdAndVersion + SkillCommandService 預檢查 + DB UNIQUE constraint 三重把關
And   T2 此 task 不驗 uniqueness — 留 T4 完整 BDD
```

### AC-3 (extended): grantAcl 後 skillRepo.save 觸發 acl_entries 行內 UPDATE（不 DELETE 子表）
```gherkin
Given Skill skill1 已存在於 skills 表 aclEntries = []
When  skill.grantAcl(...) + skillRepository.save(skill) 在 @Transactional 內
Then  PostgreSQL 收到的 SQL 為 1 條 UPDATE skills SET acl_entries = ?, updated_at = ?, version = version + 1 WHERE id = ? AND version = ?
And   無 DELETE FROM acl_entries 或 DELETE 子表 SQL（驗證 @MappedCollection delete-and-reinsert 雷被避開）
And   acl_entries jsonb 內容 = '["user:alice:read"]'
And   event_publication 表新增 SkillAclGrantedEvent listener row（async）
```

## Implementation Notes

**Skill.java 加方法**：
```java
public void suspend(SuspendCommand cmd) {
    this.status = this.status.suspend();   // state machine guard via SkillStatus enum
    this.updatedAt = Instant.now();
    registerEvent(new SkillSuspendedEvent(id, cmd.reason(), cmd.suspendedBy()));
}

public void reactivate(ReactivateCommand cmd) {
    this.status = this.status.reactivate();
    this.updatedAt = Instant.now();
    registerEvent(new SkillReactivatedEvent(id, cmd.reason()));
}

public void grantAcl(GrantAclCommand cmd) {
    var entry = entryString(cmd.type(), cmd.principal(), cmd.permission());
    if (aclEntries.contains(entry)) {
        throw new IllegalStateException("ACL entry already exists: " + entry);
    }
    aclEntries.add(entry);
    this.updatedAt = Instant.now();
    registerEvent(new SkillAclGrantedEvent(
            id, cmd.type(), cmd.principal(), cmd.permission(), cmd.grantedBy()));
}

public void revokeAcl(RevokeAclCommand cmd) {
    var entry = entryString(cmd.type(), cmd.principal(), cmd.permission());
    if (!aclEntries.remove(entry)) {
        throw new IllegalStateException("ACL entry not found: " + entry);
    }
    this.updatedAt = Instant.now();
    registerEvent(new SkillAclRevokedEvent(
            id, cmd.type(), cmd.principal(), cmd.permission(), cmd.revokedBy()));
}

public void recordDownload(String downloadedBy /* unused — kept for symmetry */) {
    this.downloadCount++;
    this.updatedAt = Instant.now();
    registerEvent(SkillDownloadedEvent.of(id, latestVersion));   // factory 自動產 eventId
}
```

**Skill.recordVersionPublished 增強（T1 已建初版；T2 加 state machine guard）**：
```java
public void recordVersionPublished(String version) {
    // state machine guard — DRAFT → PUBLISHED OK；PUBLISHED → PUBLISHED idempotent；SUSPENDED throws
    SkillStatus next = this.status.publish();
    this.latestVersion = version;
    this.status = next;
    this.updatedAt = Instant.now();
    registerEvent(new SkillVersionPublishedFromAggregate(id, version));
}
```

注意：`SkillStatus.publish()` 已是 enum-level state machine（S018 ship），DRAFT→PUBLISHED / PUBLISHED→PUBLISHED idempotent / SUSPENDED throws — 直接呼叫即用。

**aclEntries 是 mutable List**：
- 初始化由 `Skill.create()` 設為 `new ArrayList<>()`（mutable）
- `grantAcl` / `revokeAcl` 直接 `add` / `remove`
- `getAclEntries()` 回 `List.copyOf(aclEntries)` 防外部 mutate

## Anti-Patterns / Pitfalls

- **不要**為 grantAcl / revokeAcl 加同步 ACL pattern 驗證（type ∈ user/role/group、permission ∈ read/write/...）— S016 既有設計把這層驗證放在 Controller / Command 層，aggregate 只驗業務不變量（無重複 / 必須存在）
- **不要**讓 recordDownload 直接 update latestVersion — downloadCount 增量是 atomic state mutation；版本資訊 via 既有 latestVersion 欄位
- 注意 ArrayList vs List.of：T1 `Skill.create()` 用 `new ArrayList<>()`（mutable）；如果 T1 寫成 `List.of()` immutable 會在 grantAcl `.add()` 拋 UnsupportedOperationException — T2 第一步驗 T1 的 init 正確
- `getAclEntries()` 必須回不可變 view（`List.copyOf(aclEntries)`），避免外部 mutate aggregate state 繞過 grantAcl 不變量檢查

### Notes / 觀察

1. **方法命名 `record*` 前綴 vs spec §4.1 final design**：spec 設計最終名為 `suspend / reactivate / grantAcl / revokeAcl`，但 v1.5.0 既有 deprecated method 同簽名（return event）— Java 不允許 same parameter signature 同名 method 不同 return type。T2 採 `record*` 前綴避免衝突；T4 改 SkillCommandService 後 `legacy*` rename + 重新命名 `record*` 為 spec 最終名。`recordDownload` 與 `recordVersionPublished` 是 spec 設計直接同名（無衝突）。

2. **AC-3 extended (one-row UPDATE，無 DELETE 子表) 暫未驗證**：原 task plan 含 `SkillAclSqlBehaviourTest` Spring Boot 整合測試；本 T02 跳過該 test，理由：
   - 「不 DELETE 子表」由 design 保證（aclEntries 為 `List<String>` `@Column("acl_entries")` JSONB 欄位，非 `@MappedCollection` — Spring Data JDBC 不會發 DELETE 子表 SQL）
   - SQL 行為驗證需 ProxyDataSource 或 pg_stat_statements，引入測試複雜度遠超 T2 scope
   - T1 的 Spring Boot test 已驗 cross-aggregate save → outbox path；T4 的 SkillCommandService integration test 會涵蓋 grantAcl/revokeAcl 端到端
   - 建議 T6 API contract test 階段補一個 grantAcl + 觀察 SQL count 的 lightweight test（若需）

3. **`recordDownload` 無 downloadedBy 參數**：spec §4.1 設計含 `downloadedBy` 但既有 `SkillDownloadedEvent` record 無此 field（只有 aggregateId / version / eventId）。實作維持與既有 event 一致；audit downloadedBy 待 T5 AuditEventListener 階段評估（可從 SecurityContext 在 listener 端取得，不需動 event payload）。

4. **getAclEntries() unmodifiable view**：spec §4.1 line 510 `getAclEntries() { return List.copyOf(aclEntries); }` 設計保留；外部 mutate 拋 UnsupportedOperationException — T2 加 unit test 覆 AC-8 corner case。

5. **Skill 內部 `aclEntries` 必須 mutable ArrayList**：T1 `Skill.create()` 已用 `new ArrayList<>(...)` 初始化（mutable）；T2 `recordAclGranted` `.add()` 與 `recordAclRevoked` `.remove()` 依賴此特性。Spring Data JDBC `StringListJsonbConverter.Reading` 透過 Jackson `readValue` 回 `ArrayList`（mutable）— 從 DB 載入後仍可 mutate。✓
