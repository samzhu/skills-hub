## Implementation Notes

### 5B.1 AuditEventListener idempotency 設計

選項：
- **A (推薦)**: Atomic INSERT `WHERE NOT EXISTS` — 每 listener method 內 SQL pattern：
  ```sql
  INSERT INTO domain_events (id, aggregate_id, aggregate_type, event_type, payload, sequence, occurred_at, metadata)
  SELECT ?, ?, 'Skill', ?, ?::jsonb, COALESCE((SELECT MAX(sequence) FROM domain_events WHERE aggregate_id = ?), 0) + 1, ?, '{}'::jsonb
  WHERE NOT EXISTS (
      SELECT 1 FROM domain_events
       WHERE aggregate_id = ? AND event_type = ? AND payload->>'<distinguishing-key>' = ?
  )
  ```
  Race-safe；多 listener 同 aggregate 並發時透過 SERIALIZABLE 或 FOR UPDATE 處理
- **B**: 加 `event_id` field 至所有 event records；listener INSERT with `ON CONFLICT (event_id) DO NOTHING`
  需修改 9 個 event records；schema migration 加 `domain_events.event_id UNIQUE` column
- **C**: 用 SkillVersionPublishedEvent / SkillDownloadedEvent 既有 `sourceEventId` / `eventId` 為 dedup key；其他 event types 加類似欄位

選 A（最不侵入；既有 schema 不變）。

### 5B.2 SkillCommandService 縮減 final form

```java
@Transactional
public String createSkill(CreateSkillCommand cmd) {
    var skill = Skill.create(cmd);
    skillRepo.save(skill);
    return skill.getId();
}

@Transactional
public void publishVersion(PublishVersionCommand cmd) {
    if (skillVersionRepo.existsBySkillIdAndVersion(cmd.skillId(), cmd.version())) {
        throw new VersionExistsException("Version " + cmd.version() + " already exists");
    }
    var skill = skillRepo.findById(cmd.skillId()).orElseThrow();
    skill.recordVersionPublished(cmd.version());
    skillRepo.save(skill);
    skillVersionRepo.save(SkillVersion.publish(cmd));
}

// suspend / reactivate / grantAcl / revokeAcl 各 3 行 orchestration（同 spec §2.7 設計）
// 移除 saveDomainEventOnly + nextEventSequence helpers
// 移除 DomainEventRepository eventStore field
```

## Anti-Patterns / Pitfalls

- **不要**保留任何 read-model alias / facade — 整檔刪除
- **不要**在 AuditEventListener 內 reflection 反射 event 物件 — 顯式 typed listener method per event class，payload 顯式建 Map
- **不要**讓 AuditEventListener idempotency 寫法依賴 transaction-scoped state — 必須 atomic SQL（DB-level guarantee）
- **小心** sequence 計算 race — 並發 commands on same aggregate 可能 race；用 SERIALIZABLE 或 SELECT...FOR UPDATE 或 PostgreSQL advisory lock
- **小心**：T5B 移除 saveDomainEventOnly 後，所有 sync 假設 eventStore 立即可見的 tests 都會 break；逐一 audit + Awaitility wrap
