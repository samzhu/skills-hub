**設計微調（spec §2 drift）**
- AnalyticsPage 不改：page 已只顯示平台 aggregate（無個人資料），加 anon CTA 反而要新增個人 stats UI（scope creep）。已記入 §7 待補。
- `fetchMe` 加嚴格 shape check（`typeof obj.sub !== 'string' → null`）：避免 wildcard fetch mock 回 `[]` 或 `{}` 時 `user.sub` undefined → AuthArea charAt(0) throw（cascade fail 18 cases）。本身也是 anti-pattern 防線：malformed JSON 視同 anonymous，不破 UI。
- AuthGatedButton 對 loading state 的處理：直接 redirect login（不 disable button）。理由寫進 component Javadoc：用戶 race click 時若實際已登入，OAuth provider 認 session 會短路回原頁面；功能語義不破。

**Lessons**
- 任何用 `AuthGatedButton` 的 page test 都要先等 useAuth fetchMe resolve（觀測 AppShell `Open user menu` button 出現）再 click，否則 loading 視同 anonymous → 走 login() → modal 不開。Test fixture pattern 已在 RequestBoardPage AC-16 落地，後續新測比照。
