## BDD

```gherkin
Given T02 已產 useAuth() hook 與 AuthArea component
And   7 個頁面（PublishPage / CollectionsPage / RequestBoardPage /
      FlagsQueuePage / MySkillsPage / AnalyticsPage / NotificationsPage）
      都會呈現「未登入時不該執行的 action 或 personal content」
And   AC-2 user 訪問 /publish 應看到完整表單（不被 redirect）

      /oauth2/authorization/skillshub?returnTo=/publish

             - extends shadcn ButtonProps
             - onClick 內部 hook：if useAuth().status === 'authenticated'
               → 呼叫傳入的 onClick；else → useAuth().login(location.pathname
               + location.search)
             - loading 狀態維持 enabled（避免閃爍）
        2. PublishPage.tsx：Submit 按鈕改 <AuthGatedButton onClick={submit}>
        3. CollectionsPage / RequestBoardPage / FlagsQueuePage：所有
           「需要 identity」的 action button 改 AuthGatedButton；read-only
           list 內容不動
        4. MySkillsPage / AnalyticsPage / NotificationsPage：useAuth 狀態
           分支：
             - anonymous → 顯示 CTA card「登入後可查看自己的 X」+ <Button>
               登入</Button>（onClick → useAuth.login()）
             - authenticated → 既有頁面內容

Then  AC-2 anonymous user 訪問 /publish 看到完整表單（與已登入時相同
      渲染；只 Submit 按鈕點擊行為不同）
  And AC-3 anonymous user 點 Submit → location 轉 /oauth2/authorization/
      skillshub?returnTo=/publish（spy window.location 或測試 dispatch）
  And anonymous 點 /my-skills /analytics /notifications → 看 CTA 提示
      （非 redirect、非 throw）
```
