**設計差異紀錄（與 spec §4.6 對齊）**
- spec §4.6 的 yaml 用 `client-id: ${spring.security.oauth2.client.registration.skillshub.client-id}` 自我引用 — 不會解析（self-reference loop）。實作改為 yaml 完全不寫 client-id/client-secret，由 Cloud Run env var 直接綁定（Spring relaxed binding）。功能等價但更簡潔，無 secret leak 風險。spec §7 會記入 design drift。
