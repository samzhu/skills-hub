# S139-T04: LAB OAuth 部署 + E2E smoke test

## Spec
S139 — Login UI + Lazy Auth Gate

## BDD

```gherkin
Given T01-T03 完成 — code 改動 ready in image
And   LAB Cloud Run 既有 deployment（revision skillshub-00006，per S139 前的
      部署）`skillshub.oauth.issuer-uri` 為空字串、無 OAuth client 設定
And   Google Cloud Console OAuth client 已註冊 localhost redirect URI
      （S134 留下），無 LAB redirect URI

When  完成下列部署 + 設定動作：
        1. lab.yaml 加 spring.security.oauth2.client.registration.skillshub
           完整 block + provider.skillshub.issuer-uri =
           https://accounts.google.com（per spec §4.6）
        2. 重 build image（gcloud builds submit）+ push secret 新版（
           gcloud secrets versions add skillshub-app-config-lab）
        3. service.rendered.yaml（在 temp/）加 2 個 secretKeyRef env：
             - spring.security.oauth2.client.registration.skillshub.client-id
             - spring.security.oauth2.client.registration.skillshub.client-secret
        4. 部署人員手動：
             a. Google Cloud Console 加 LAB redirect URI
                https://<lab-cloud-run-url>/login/oauth2/code/skillshub
             b. 建 2 個 Secret Manager secret：
                  skillshub-oauth-client-id
                  skillshub-oauth-client-secret
                授權 SA accessor
             c. gcloud run services replace 重 deploy
        5. 瀏覽器 smoke test：
             a. 訪問 https://<lab-cloud-run-url>/browse → 200 + 看到 skill list
                + AppShell「登入」按鈕（AC-1）
             b. 點「登入」→ 302 到 accounts.google.com → 完成 Google 登入
                → 302 回 /login/oauth2/code/skillshub → session 建立
             c. 訪問 /api/v1/me → 200 + email/sub/name claim
             d. AppShell 顯示 avatar dropdown（AC-4）
             e. 訪問 /publish → 看到表單（AC-2）→ 試填 + Submit → 上傳成功
                （已登入下走正常 flow）
             f. 點 dropdown 登出 → AppShell 變回「登入」按鈕（AC-5）

Then  AC-8 evidence-only AC：browser screenshot + curl /api/v1/me 200 logs
      由部署人員附在 spec §7
```

## Target Files

**修改**
- `backend/config/application-lab.yaml`（per spec §4.6）— 加 OAuth client
  registration block + 改 `skillshub.oauth.issuer-uri` 從空變
  https://accounts.google.com
- `temp/service.rendered.yaml`（部署 spec；不在 git，部署人員自行同步）
  — 加 2 個 secretKeyRef env

**部署人員手動**（不 commit）
- Google Cloud Console redirect URI 註冊
- gcloud secrets create skillshub-oauth-client-id / -secret + IAM
- gcloud builds submit → 拿新 tag
- service.rendered.yaml image tag 換成新 tag
- gcloud run services replace
- gcloud secrets versions add skillshub-app-config-lab --data-file=
  backend/config/application-lab.yaml（推 secret v3 對齊新 yaml）
- 瀏覽器 smoke test

## Verification

1. `grep -A 10 "registration:" backend/config/application-lab.yaml` → 看到
   `skillshub:` block + scope `openid,email,profile` + redirect-uri
   `{baseUrl}/login/oauth2/code/skillshub`
2. 部署後：`curl https://<lab>/oauth2/authorization/skillshub` → HTTP/2 302
   + `location: https://accounts.google.com/o/oauth2/v2/auth?...`
3. 部署後：手動瀏覽器跑 BDD When 5.a–5.f；附 console.log 或 screenshot
4. `curl https://<lab>/api/v1/me` （帶有效 session cookie） → 200 +
   `{"sub": "...", "email": "...", "name": "...", ...}`
