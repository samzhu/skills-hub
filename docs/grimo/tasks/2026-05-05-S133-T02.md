# S133-T02: Frontend MarkdownActionMenu dropdown + Safari-safe clipboard hook

## Spec
S133 — Skill Markdown Export 前端 dropdown UI + clipboard

## BDD

```
Given user 訪問 /skills/abc-123，該 skill status=PUBLISHED
When SkillDetailPage 渲染
Then 在 SkillHero 右側 button cluster 出現「Markdown ▾」trigger
And 點 trigger 展開 menu，2 個 item：「複製為 Markdown」+「開啟 Markdown」

Given user 訪問 DRAFT 或 SUSPENDED skill 詳情頁
Then「Markdown ▾」trigger 不出現

Given user 點「複製為 Markdown」
Then 走 navigator.clipboard.write([new ClipboardItem({'text/plain': Promise<Blob>})])（NOT writeText with awaited fetch — Safari transient activation 限制）
And blob 內容 = SKILL.md raw body
And sonner toast「已複製到剪貼簿」

Given user 點「開啟 Markdown」
Then 新分頁開啟 /api/v1/skills/{id}/skill.md
And anchor target=_blank rel=noopener,noreferrer
```

## Implementation Notes

### Step 1 — Install shadcn DropdownMenu
```bash
cd frontend && npx shadcn@latest add dropdown-menu
```
寫入 `frontend/src/components/ui/dropdown-menu.tsx` local wrapper（per shadcn 2026-02 changelog — `radix-ui` unified package 已在 deps，CLI 仍需寫 wrapper）。

### Step 2 — Hook
`hooks/useCopySkillMarkdown.ts`：
- `prefetch()` — `onOpenChange` 開啟時呼叫；`useRef<string>` cache；fail silently
- `copy()` — 必須**從 click handler 同步呼叫到 `navigator.clipboard.write`**；cache hit 走 `Promise.resolve(new Blob(...))`，cache miss 走 `fetch().then(text).then(blob)` 但 promise 同步交給 ClipboardItem
- 不用 `await fetch().then(writeText)`（Safari 會 NotAllowedError）

### Step 3 — 元件
`components/MarkdownActionMenu.tsx`：
- `<DropdownMenu onOpenChange={(open) => open && prefetch()}>`
- Trigger: 「Markdown」+ ChevronDown icon, `aria-label="Markdown 操作"`
- 2 items: Copy（onClick → copy().then(toast.success)） / Open（asChild + `<a target=_blank>`）

### Step 4 — Page mount
`pages/SkillDetailPage.tsx` `SkillHero` line ~272 button cluster 加：
```tsx
{skill.status === 'PUBLISHED' && (
  <MarkdownActionMenu skillId={skill.id} />
)}
```
位置：`<SubscribeButton>` 與「下載」anchor 之間（hero row 右側）。

- AC-7：assert `clipboard.write` 被呼叫且傳入的 ClipboardItem 內 blob.text() === fixture

- AC-8：click「開啟 Markdown」→ assert anchor 有 `target=_blank` + `href` 對齊 `/api/v1/skills/{id}/skill.md`

**`SkillDetailPage.test.tsx`** (modify existing):
