## Execution Steps

```bash
# Step 1：確認 application-real-oauth.yaml 已設好 Google credentials
# （前 session 已完成，backend 已用 Google config 啟動）

# Step 2：確認 backend 是否仍在跑（real-oauth profile）
lsof -nP -iTCP:8080 -sTCP:LISTEN
# 若沒在跑：
cd backend
SPRING_PROFILES_ACTIVE=local,dev,real-oauth ./gradlew bootRun > /tmp/skillshub-real-oauth.log 2>&1 &

# Step 3：AC-2 — 驗 discovery 302（若前 session 沒記錄，再跑一次）
curl -s -i http://localhost:8080/oauth2/authorization/skillshub | head -20
# 預期：HTTP/1.1 302 + Location: https://accounts.google.com/o/oauth2/v2/auth?...
# 觀察：scope=openid%20email%20profile + code_challenge + nonce + state

# Step 4：AC-3 — 完整 Google login flow（user 自己的瀏覽器！不走 Chrome MCP）
# 1. 開無痕 → http://localhost:8080/oauth2/authorization/skillshub
# 2. Google 登入頁 → 選擇 Google 帳號 → 允許授權
# 3. 預期 302 回 callback → land 在 http://localhost:8080/

# Step 5：AC-4 — claim shape dump
# 同瀏覽器 → http://localhost:8080/auth-debug 或 GET /api/v1/dev/auth-debug
# 把 JSON 整段複製貼到 spec §7
# 預期：id_token_claims.iss = https://accounts.google.com, email/name/picture 有值
# 預期：access_token_claims = {"error": "..."} 或 base64 亂碼（Google opaque，正常）

# Step 6：AC-5 — N/A，直接在 spec §7 標記說明

# Step 7：AC-6 final regression — 切回預設 profile
# Ctrl-C 或 kill 後改跑 ./gradlew bootRun（無 profile 變數）
curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/api/v1/skills   # 預期 200
curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/api/v1/dev/auth-debug  # 預期 404

# Step 8：檢查 backend log（若 AC-3 成功）
grep -E "login/oauth2/code|OAuth2Login|token|OAuth2" /tmp/skillshub-real-oauth.log | tail -20
```
