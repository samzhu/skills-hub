# S133-T01: Backend SkillMarkdownController alias

## Spec
S133 — Skill Markdown Export 對 agent 友善的 raw md alias

## BDD

```
Given skill `id=abc-123` 為 PUBLISHED PUBLIC，最新版 zip 含 SKILL.md
When agent 對 GET /api/v1/skills/abc-123/skill.md 發 request
Then 回 HTTP 200
And Content-Type: text/markdown
And Body 等於 zip 內 SKILL.md 完整內容
And Cache-Control header 含 public, max-age=60

Given skill PRIVATE 無 ACL grant 給 anonymous
When anonymous 對 /skill.md 發 request
Then 回 401（per Spring Security ExceptionTranslationFilter on AnonymousAuthenticationToken）

Given skill SUSPENDED
When 發 request
Then 回 403 SKILL_SUSPENDED（既有 SkillSuspendedException → GlobalExceptionHandler mapping）

Given skill id 不存在
When 發 request
Then 回 404
```

## Implementation Notes

- 新 controller `SkillMarkdownController` 委派 `FileBrowserService.readFile(id, "SKILL.md")` — 不重複 zip-slip / size / SUSPENDED 防禦邏輯
- `@PreAuthorize("hasPermission(#id, 'Skill', 'read')")` 對齊 `SkillQueryController.getById` (S122 既驗 pattern)
- `@PathVariable UUID id` — 對齊 S126 invalid format → 400 fast-fail（避開 PreAuthorize 走 fail-secure 誤判）
- `Cache-Control: public, max-age=60` 透過 `CacheControl.maxAge(Duration.ofSeconds(60)).cachePublic()`
- Test 走 `@WebMvcTest(SkillMarkdownController.class) extends WebMvcSliceTestBase`（per S025b 既驗 slice base class）
  - mock `FileBrowserService` 回 `FilePreview(bytes, "text/markdown")`
  - 用 `.with(jwt())` post-processor 模擬 authenticated；anonymous 不加 jwt
  - mock `PermissionEvaluator` 對應 4 個情境（grant=200 / no-grant=401 anonymous / no-grant=403 authenticated / suspended=throw → 403）
- AC tagging：每 test method `@DisplayName("AC-1: ...")` + `@Tag("AC-1")`
