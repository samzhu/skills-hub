# S024-T03: SkillVersion 獨立 aggregate 完整化 + repository derived queries + attachRiskAssessment

## Spec
S024 — Skill State-Based Aggregate Migration

## Goal
T1 已建 SkillVersion 最小骨架（publish factory）。T3 補完獨立 aggregate 完整功能：`attachRiskAssessment` 充血方法 + `SkillVersionRepository` 補齊 derived queries（findBySkillId / findBySkillIdAndVersion / `@Modifying @Query updateRiskAssessment` for ScanOrchestrator 改造的支援）。

關鍵不變式：SkillVersion 透過 `String skillId` 引用 Skill（plain FK 字串，**不**用 `@MappedCollection` 也**不**用 `AggregateReference`，避免 delete-and-reinsert 風險）。跨 aggregate 一致性（同 skill 不能有重複 version）由 application service `SkillCommandService.publishVersion` 在 `@Transactional` 內 `existsBySkillIdAndVersion` 預檢 + DB UNIQUE constraint 兜底（兩道防線；T4 完成 service 端 orchestration）。

## Implementation Notes

**SkillVersion.java（補完）**：
```java
@Table("skill_versions")
public class SkillVersion extends AbstractAggregateRoot<SkillVersion> {

    @Id private String id;
    private String skillId;
    private String version;
    @Column("storage_path") private String storagePath;
    @Column("file_size") private long fileSize;
    @Column("frontmatter") private Map<String, Object> frontmatter;
    @Column("risk_assessment") private Map<String, Object> riskAssessment;   // null until scan
    @Column("published_at") private Instant publishedAt;
    @Column("allowed_tools") private List<String> allowedTools;

    private SkillVersion() {}                   // for Spring Data JDBC

    public static SkillVersion publish(PublishVersionCommand cmd) {
        var sv = new SkillVersion();
        sv.id = UUID.randomUUID().toString();
        sv.skillId = cmd.skillId();
        sv.version = cmd.version();
        sv.storagePath = cmd.storagePath();
        sv.fileSize = cmd.fileSize();
        sv.frontmatter = cmd.frontmatter();
        sv.riskAssessment = null;
        sv.publishedAt = Instant.now();
        sv.allowedTools = parseAllowedTools(cmd.frontmatter());
        sv.registerEvent(SkillVersionPublishedEvent.of(
                sv.skillId, sv.version, sv.storagePath, sv.fileSize, sv.frontmatter, sv.allowedTools));
        return sv;
    }

    public void attachRiskAssessment(Map<String, Object> assessment) {
        this.riskAssessment = assessment;
        registerEvent(new SkillRiskAssessedEvent(
                skillId, version, (String) assessment.get("level"), assessment.get("findings")));
    }

    // accessors: id() / skillId() / version() / storagePath() / fileSize() /
    //           frontmatter() / riskAssessment() / publishedAt() / allowedTools()

    private static List<String> parseAllowedTools(Map<String, Object> frontmatter) { /* 復用 SkillCommandService.parseAllowedTools 邏輯 */ }
}
```

## Anti-Patterns / Pitfalls

- **不要**在 SkillVersion 加 `@MappedCollection` 引用 Skill（per ADR-002 §2.3 + deepwiki §2.1 — delete-and-reinsert）
- **不要**用 `AggregateReference<Skill, String>`（同樣框架介入子集合 lifecycle 風險）— 用 plain `String skillId`
- **不要**保留 `SkillVersionReadModelRepository.updateRiskAssessment` 的 @Modifying @Query 模式 — 該 SQL 直接寫表繞過 @DomainEvents publish；ScanOrchestrator 改造（T5）會用 `repo.findById + attachRiskAssessment + save` 走 aggregate event publish path
- 注意 `Map<String, Object>` JSONB converter 已在 JdbcConfiguration 註冊（Map → PGobject）— frontmatter / riskAssessment 欄位免額外設定；List<String> allowedTools 用 StringListJsonbConverter 同樣已註冊
- 既有 `SkillVersionPublishedEvent.of()` factory 含 `sourceEventId` 自動產生 — 維持語義；retry 時不應改變 sourceEventId（既 S023 idempotency key）

### Notes / 觀察

1. **AC-7 部分覆蓋**：DB UNIQUE constraint 兜底已驗（直接 bypass service 寫入 → DataIntegrityViolationException）；service 層 `existsBySkillIdAndVersion` 預檢拋 `VersionExistsException` 由 T4 SkillCommandService 改造完整覆蓋（剩 AC-7 full）。

2. **`hasRiskAssessmentFromEvent` JSONB 查詢驗證**：`risk_assessment->>'sourceEventId' = :sourceEventId` PostgreSQL JSONB 文字比對運作正常。null risk_assessment（未掃描版本）→ 子查詢回 false。三個 corner case 全綠（命中 / 不同 sourceEventId / 不同 version）。

3. **`attachRiskAssessment + save` UPDATE path 驗證**：`@PersistenceCreator private SkillVersion()` 載入既有 row 後 isNew=false → repo.save 走 UPDATE path（per Persistable.isNew() spec）。Map<String, Object> JSONB 透過既有 `MapToPGobjectConverter` 雙向序列化正常。

4. **Edit 失誤修復**：第一次 Edit SkillVersion.java 時 @Override 與 attachRiskAssessment 方法位置錯亂導致 compile error（@Override 落在新方法上）；修正後 PASS。

5. **Compile warnings**：仍有「uses or overrides a deprecated API」warnings（既有 Skill 與 SkillVersionPublishedEvent.of factory 等 @Deprecated 標記）— T4/T5 階段隨呼叫端改造逐步消除。

6. **POC outbox 行為延伸**：T03 SkillVersionRepositoryTest 中 `attachRiskAssessmentAndSave` 觸發 SkillRiskAssessedEvent register + publish 至 outbox — 但 T1-T3 期間無 listener 訂閱此事件（T5 加 AuditEventListener 訂閱），outbox row count = 0。屬預期行為，不影響 test。
