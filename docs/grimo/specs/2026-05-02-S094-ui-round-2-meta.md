## §2 Approach

### Sub-spec sequencing

| # | Spec | Page / Topic | Estimate | Backend impact | PRD link |
|---|------|--------------|----------|----------------|----------|
| 1 | **S094c** | Empty State Collection | XS (5-6) | 0 | UX polish across 4 page |
| 2 | **S094d** | Docs Walkthrough | XS (5-6) | 0 | P2 entry point |
| 3 | **S094a** | My Skills (Author Dashboard) | M (12-13) | `?author=` filter + 30d trend | P6 SBE 「作者查看自己的數據」 |
| 4 | **S094b** | Semantic Search Results | M (12-13) | LLM intent summary endpoint | P5 polish + concept chips |

**Order rationale**: `size 小先`（C/D 兩個 XS pure-FE 快速 ship 增加可重用 EmptyState / Markdown components）→ `共享 component 抽取先`（C 提供 EmptyState 給 A 用）→ `foundation 先`（A 落 author scope，B 才能 reuse SearchResultCard 模板）。

### Reuse posture（per Maximize framework reuse）

既有 components 可直接 reuse:
- `AppShell` — page chrome（4 個 sub-specs 全用）
- `IconTile` — category tint（A: table column / D: bundle tree icon）
- `MetricCard` — 4-metric strip（A: hero metrics）
- `RiskBadge` — risk pill（A: table row / B: result card）
- `SkillCard` — search result template（B: extend with similarity score + why-match）
- `BeamFrame` — primary CTA wrapper（C: empty-state primary action / D: upload bundle CTA）
- `SearchBar` — header（B: with semantic dot variant）

新增 components（最小集，每個 sub-spec 提一個）:
- `EmptyState` (C) — 4-tone variant, reusable across pages
- `MarkdownDoc` (D) — markdown render wrapper, lightweight (用 `react-markdown@latest`)
- `Sparkline` (A) — inline mini-chart for table rows
- `ConceptChip` / `RefineChip` (B) — chip with × / ↗ affordance

不重新發明 wheel：
- Table component → 用 native `<table>` + Tailwind class（既有 `MetricCard` 是 hairline border 模式；同 pattern 套 table）
- Tabs → 用 既有 `SkillDetailPage` 的 tab pattern（同等不抽組件，避免 over-engineer）
- Stepper（D 的 prev/next） → inline `<a>` 即可

### Confidence classification (Research Sufficiency Gate)

| Decision | Confidence | Source |
|----------|------------|--------|
| 既有 5 components reusable as-is | **Validated** | source 已讀 (S085-S088 ship) |
| `react-markdown` 適合 D 的 docs render | **Validated** | npm-registered, MIT, React 19 compatible |
| `?author=` filter 加進 `GET /skills` 為小改動 | **Validated** | `SkillQueryService.search` 已支援 keyword/category 同 pattern |
| 30d per-skill trend 需新 endpoint | **Validated** | `download_events` 表已有 (skill_id, occurred_at)；新增 `/skills/{id}/analytics/trend?days=30` |
| LLM intent summary 設計 | **Hypothesis** | Spring AI 既有，但「intent summary + concept extraction」是新 chat call；prompt 設計 + cost control 待 POC |

**S094b POC required**: intent summary 的 prompt 設計與 latency 預算（target ≤2s P95）；否則 fallback 為 client-side "echo your query" 而非真 LLM intent extraction.

## §4 Sub-spec snapshots

### S094a — My Skills (Author Dashboard)

**Mockup**: `my_skills_author_dashboard.html`（README ll.69-87）
**Route**: `/my-skills`
**Estimate**: M (12-13)

**Layout**:
- Hero: `7 skills you've published as platform-team` + Publish CTA (BeamFrame)
- 4 summary MetricCards: Total skills (status breakdown) / Total downloads (week delta) / Avg rating / Open flags (red delta if >0)
- Tabs: All / Published / Pending review / Drafts
- Table columns: Skill (name+author+meta) / Status (pill) / Downloads 30d (number + Sparkline) / Latest version + date / Rating / ⋯
- Special row treatments: Pending row amber tint / Draft row 0.85 opacity gray

**Backend touch**:
- Add `?author=` to `GET /skills` (query param only; existing pageable preserved)
- New `GET /api/v1/skills/{id}/analytics/trend?days=30` (per-skill download bucket array for sparkline)

**FE new components**: `Sparkline.tsx`

### S094b — Semantic Search Results

**Mockup**: `semantic_search_results_page.html`（README ll.111-133）
**Route**: `/search?q=...&mode=semantic`
**Estimate**: M (12-13)

**Layout**:
- SearchBar with semantic mode dot pulse (purple #7F77DD)
- AI intent summary card (purple #EEEDFE bg) + concept chips (× to remove)
- Results meta: `ranked by semantic similarity · embeddings via Gemini · 7 skills`
- Top match: gradient bg + star badge + 0.94 score
- Each result card: description with concept highlights + why-match reasoning (gray sub-card) + similarity score with mini bar
- Refine chips: `Only low-risk skills` / `For Kubernetes specifically` / etc.

**Backend touch**:
- New `POST /api/v1/search/intent` (input: query string; output: `{summary, concepts: string[]}`); LLM call (Gemini); 5-min response cache
- Reuse `GET /api/v1/search/semantic` for ranking

**POC required**: intent summary prompt + latency budget; fallback to client-side echo if POC HALT.
**FE new components**: `ConceptChip.tsx` / `RefineChip.tsx` / `IntentSummaryCard.tsx`

### S094c — Empty State Collection

**Mockup**: `empty_state_collection_four_tones.html`（README ll.221-243）
**Estimate**: XS (5-6)

**4 Variants**:
1. **Fresh deployment (Seeding)** — for HomePage when 0 skills exist; purple eyebrow + dashed ghost cards + `Publish the first skill` CTA
2. **New author (Invitational)** — for `/my-skills` when author has 0 skills; horizontal 4-step flow + `Get the starter template` link
3. **No search results (Redirecting)** — for `/search?q=...` when 0 results; echo query + 4 suggestion cards
4. **Review queue all clear (Celebratory)** — admin context (defer to S094e B6); for now 留 hook 給 future spec

**FE new components**: `EmptyState.tsx` with `tone: 'seed' | 'invite' | 'redirect' | 'clear'` prop

**Integrations**:
- HomePage when `total === 0`
- Future: MySkills when `mySkills.length === 0` (reuse via S094a)
- Future: SearchResults when `results.length === 0` (reuse via S094b)

### S094d — Docs Walkthrough

**Mockup**: `docs_page_write_your_first_skill.html`（README ll.245-279）
**Route**: `/docs/your-first-skill`
**Estimate**: XS (5-6)

**Layout**:
- Top app shell with Docs nav active
- Left sidebar IA: Getting started / Reference / Publishing / API & webhooks
- Main column: Meta row (Updated · 5 min read · agentskills.io v1.2 · Edit on GitHub) → Intro → Minimum viable skill code → Bundle tree with badges → Required fields cards → Description A/B compare → Risk tiers pills → CTA block → prev/next nav

**Content source**: Single markdown file `frontend/src/docs/your-first-skill.md` rendered via `react-markdown`. AppShell + Docs sidebar are React components; markdown is content-only.

**FE new components**: `DocsLayout.tsx` / `DocsSidebar.tsx` / `MarkdownDoc.tsx` (wrap react-markdown)
**Dependencies**: `react-markdown@latest` (small ~30KB gz)
