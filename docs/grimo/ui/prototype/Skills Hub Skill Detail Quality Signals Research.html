原本要做什麼

明細頁要讓使用者在下載前看懂三件事：SKILL.md 是否符合規格、內容是否容易執行、description 是否容易被 agent 觸發。

現在卡住哪裡

ScoreDot 只懂 0-3 分，但規格驗證回傳 0-100 分與 warnings 陣列，所以 100 分也可能顯示橘點，warnings 變成紅點。

競品學到什麼

Tessl 把 Quality 拆成 Validation / Implementation / Activation，並直接寫出 pass、warning、fail 與修正建議；GitHub、JetBrains 的 badge 也用 tooltip/text 限定意義。
