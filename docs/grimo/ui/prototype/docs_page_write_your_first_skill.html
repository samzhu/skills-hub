Write your first skill

Updated 2 weeks ago 5 min read Based on agentskills.io v1.2 Edit on GitHub

This is the walkthrough for authors who haven't published before. You'll leave with a working SKILL.md bundle and a mental model for what the auto-scanner checks at publish time. Nothing here is tied to a specific agent — the agentskills.io format works across every compatible tool.

The minimum viable skill #

A valid bundle needs exactly one file: SKILL.md with YAML frontmatter and a markdown body. Everything else — scripts/, references/, assets/ — is optional. Start here:

date-formatter/SKILL.md Copy
--- name: date-formatter description: Convert between common date formats. Use when the user pastes a date string and asks to reformat, or when producing structured date output. version: 1.0.0 license: MIT --- # Date formatter Invoke this skill whenever the user needs to convert a date between formats. Accepts free-form input like "Jan 3, 2026" or "2026-01-03T08:00Z" and produces any ISO 8601 or RFC 2822 variant. ## When to use - The user pastes a date and asks to reformat it - Output needs a specific timezone or locale
Only name and description are required. Everything else is optional. This bundle passes validation and auto-publishes at low risk because there's no scripts/ folder.

The bundle #

Once you outgrow a single file, organize your bundle into the four conventional folders. Only SKILL.md is required; the rest are loaded on demand by the agent or used during runtime.

date-formatter/ tree
date-formatter/ ├── SKILL.mdrequired ├── scripts/triggers scan │ └── normalize.sh ├── references/optional │ └── iso-8601.md └── assets/optional └── examples.yaml

Required fields #

name
required string ≤ 64 chars

Lowercase, hyphen-separated. Unique within your org — publishing a second skill with the same name will bump the version instead of creating a new one. Used in URLs, CLI commands, and skill imports. Example: docker-compose-helper

description
required string ≤ 1024 chars

One concise paragraph covering what the skill does and when an agent should invoke it. Semantic search indexes against this field — write it like a trigger condition, not a marketing blurb. See the next section for examples.

Optional fields: version, author, license, compatibility, metadata, allowed-tools. See the full frontmatter reference.

Writing a description that works #

Semantic search embeds your description with Gemini and matches user queries against the vector. The model picks up on concrete verbs, trigger conditions, and domain nouns — it ignores marketing adjectives. Two examples from the same skill:

Indexes well

Generate and validate docker-compose files with multi-service awareness. Use when the user edits compose.yml or needs a new stack.

Concrete verbs ("generate", "validate"), domain noun ("compose.yml"), trigger condition ("when the user edits"). Matches queries about Docker, orchestration, YAML, and stacks.

Too vague

The best helper for Docker stuff — easy to use, powerful, and works with any deployment scenario you throw at it.

Adjectives without content. No verbs the embedding can anchor on, no trigger condition. "Docker stuff" is too broad to rank above unrelated skills.

What triggers each risk tier #

Every publish runs through the auto-scanner. The tier determines whether the skill goes live immediately or waits for human review.

Low

Publishes immediately. No scripts/ folder, no external URLs, clean frontmatter. This is the default path for most well-written SKILL.md bundles.

Medium

Publishes with a warning badge. Scripts exist but no dangerous patterns detected — up to 3 external URLs, all on the trusted-source registry. Consumers see the tier clearly in search.

High

Blocked until a reviewer approves. Triggered by rm -rf on user paths, curl | bash, reads from ~/.ssh or ~/.aws, or untrusted domains.

High-risk isn't a rejection. A reviewer will either approve it (with a note in the public audit log), ask you to fix specific findings, or reject it. Full rule catalog is at /docs/risk-tiers.

Ready to publish?

You've got a bundle, the frontmatter is clean, and you know what the scanner checks. Ship it.
