Write your first skill

A skill is a .zip bundle containing a SKILL.md file that tells your AI agent how to handle a specific task. Skills follow the open agentskills.io standard — compatible with Claude Code, Cursor, Copilot, and 25+ other tools. This guide walks you through authoring, scanning, and publishing a skill from scratch.

Minimum viable skill

The only required file is SKILL.md. A working skill looks like this:

SKILL.md--- name: date-formatter description: Convert between common date formats. Use when the user pastes a date string and asks to reformat it (e.g. ISO 8601 → US short, epoch → human-readable). version: 1.0.0 author: your-team license: MIT --- ## Instructions When the user provides a date string and asks to change its format: 1. Identify the input format (ISO, epoch, US, EU, etc.) 2. Ask the user for the target format if not specified 3. Output the converted date and explain any ambiguity
Auto-publishes at low risk This skill has no scripts/ directory, so the risk scanner will mark it Low automatically and publish it without human review.

Bundle structure

Zip the skill folder. The archive must contain SKILL.md at the root.

SKILL.md
required
The skill's instructions and metadata. This is the only file the scanner always checks.
scripts/
optionaltriggers scan
Shell, Python, or JS scripts. Presence of this folder triggers the full risk scan. See scan rules.
references/
optional
Markdown or text files the agent reads at runtime for context. No execution risk.

Required SKILL.md fields

name
requiredstring ≤ 64
Lowercase, hyphenated, globally unique within your org. Used as the install identifier: skills-hub install your-team/skill-name
description
requiredstring ≤ 1024
Plain English, one to three sentences. This is embedded for semantic search — write it for the person describing a task, not a product manager.

Writing a description that works

Your description is fed into Gemini embeddings and ranked against user queries. The difference between being found and invisible is usually here.

✓ Good
"Convert between common date formats. Use when the user pastes a date string and asks to reformat it — ISO 8601 → US short, epoch → human-readable."
Concrete verb (convert), domain nouns (ISO 8601, epoch), trigger condition (user pastes a date string)
✗ Weak
"A powerful and flexible date handling tool for all your formatting needs."
Adjectives without content. Embeddings have nothing to anchor on. Won't surface for "reformat ISO date".

Risk tiers

Every upload is scanned automatically. The result determines what happens next.

LowPublishes immediately. No scripts/ directory, or all scripts pass all checks.
MediumPublishes with a warning badge. 1–2 medium-severity findings that don't block installation.
HighBlocked until a reviewer approves, rejects, or requests changes. Common triggers: curl | bash, rm -rf, ~/.aws, ~/.ssh.
High-risk isn't a rejection A high-risk result enters the review queue. A security reviewer will approve with a note, ask for fixes, or reject. Authors are notified either way.

Publish your skill

$ zip -r date-formatter.zip date-formatter/ $ skills-hub publish date-formatter.zip ✔ Validated SKILL.md (name, description, version present) ✔ Risk scan: LOW — no scripts/ directory ✔ Published → skills-hub.io/your-team/date-formatter SkillVersionPublished event written

Ready to publish?

You've got a bundle, the frontmatter is clean, and you know what the scanner checks.
