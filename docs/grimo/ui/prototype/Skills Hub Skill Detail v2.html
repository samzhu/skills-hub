docker-compose-helper

When to use this skill

Invoke when working with docker-compose.yml files — authoring a new stack, adding services, debugging why containers fail to start, or hardening an existing compose file for production.

  • Scaffolding new multi-service stacks with sensible defaults
  • Adding healthchecks, depends_on conditions, and restart policies
  • Converting ad-hoc docker run commands into compose services
  • Reviewing compose files for rootless, resource-limit, and secret-handling issues

What it does

This skill instructs the agent to think about Docker Compose topology holistically — not just generating syntactically valid YAML, but reasoning about service dependencies, network isolation, volume mounts, and startup ordering. It brings awareness of common production pitfalls like missing healthchecks, world-writable mounts, and leaked environment secrets.

Examples

Generate a new stack

Ask the agent: "Create a docker-compose for a Next.js app with Postgres and Redis, with healthchecks and a non-root user."

Debug startup failures

Paste your compose file and describe the error. The skill guides the agent to inspect depends_on conditions, healthcheck probes, and port conflicts systematically.

Harden for production

Ask for a security review of any compose file. The skill will flag world-writable volumes, missing read_only flags, containers running as root, and exposed ports that don't need to be public.
